# Codex 内容应急箱

这是给 Codex 用的异步应急入口，只服务内容线问题，不走 OpenClaw 业务分发。

## 你怎么用

1. 直接编辑《老板给Codex.md》
2. 用自然语言写清楚你要我做什么
3. 如果有截图/日志，放到《附件/》
4. 等 Codex 把结果写回《Codex回复.md》
5. 不需要手动清空旧需求；系统会按《老板给Codex.md》的内容签名判断是不是新请求

## 规则

- 不需要写 task_id
- 不需要新建很多文件
- 每次只改《老板给Codex.md》即可
- 只建议用于内容线巡检、交付、草稿箱、图片、桥接、展示台等问题
- 若你只是查看旧回复，不会触发重复处理
- 只有当《老板给Codex.md》内容真的变了，才会被当成新请求
