# 同行资本市场内容系统｜公域品牌上下文手册

## 1. 适用范围

这份手册只约束 **公域内容**：

- 微信公众号
- 小红书
- 知乎
- X
- B站专栏
- 今日头条
- 百家号

私域维护、社群运营、后续转化跟进由真人负责，不在本轮系统自动化范围内。

但公域内容需要具备一个明确意识：

> 内容不能只追阅读量，还要给目标客户留下“值得进一步交流”的钩子。

### 1.1 当前执行口径

自 `2026-03-26` 起，公域内容采用 **7 平台分层打法**：

- 公众号：内容资产主库
- 小红书：破圈增长引擎
- 知乎：搜索承接主阵地
- X：全球信号雷达
- B站专栏：builder 社区深度补位
- 今日头条：泛流量与标题实验层
- 百家号：百度 SEO 镜像层

注意：

- 这 7 个平台都进入正式支持范围
- 但不是每个题都 7 平台齐发
- 必须先选平台束，再做平台化改写

## 2. 当前公域主定位

冷启动阶段，不把同行资本对外做成“高冷研究院”，而是先做成：

> **同行资本｜AI Agent 前沿机会雷达**

一句话解释：

> **围绕 AI Agent、自主运营、一人公司和相关关键变化，把复杂热点讲明白，把值得看的机会先筛出来。**

这里要注意：

- 我们的内容边界先钉死在 **AI 圈**
- 但不会僵化成“只讲 agent 本体”
- 模型、agent infra、AI 硬件、平台变化，只要会实质改变 agent 业务和一人公司，也都可以讲

公域不主打“我掌握最稀缺认知”，公域主打三件事：

1. 看得早
2. 筛得快
3. 讲得明白

## 3. 公域内容的四个核心任务

每一条公域内容，至少应完成下面一项，最好同时完成两项以上。

### 3.1 筛选

帮用户判断：

- 这件事值不值得看
- 它是噪音，还是值得继续跟踪的变化

### 3.2 解释

帮用户快速看懂：

- 发生了什么
- 为什么重要
- 重要性到底来自技术、产业还是商业侧

### 3.3 提示机会感

不是直接喊结论，而是让用户看到：

- 哪条链路可能在起变化
- 哪类公司、产品、场景值得继续关注
- 下一步最该看什么指标

### 3.4 留下转化钩子

公域内容的终点不是“写完就结束”，而是让一部分高价值用户愿意：

- 关注
- 收藏
- 私信
- 评论互动
- 进一步交流

## 4. 主战场与邻接领域

同行资本公域内容的**主战场**是：

- AI Agent 产品、能力与 workflow
- 一人公司 / 超级个体 / agent 自主运营业务
- skill / workflow / stack 拆解
- 真实业务案例拆解
- 同行资本自己的 build in public

可以灵活覆盖的**邻接领域**包括：

- 会改变 agent 能力边界的模型重大更新
- 会改变 agent 落地效率的 agent infra 变化
- 会改变 agent 使用场景和交互体验的 AI 硬件 / AI 终端
- 会改变 agent 创业、分发、收费、部署的生态和平台事件

核心纪律不是“比例”，而是：

> **所有邻接内容，最后都要回扣到 agent、自主运营、一人业务或 workflow 的真实含义。**

## 5. 目标用户分层

## 5.1 第一层：泛科技关注人群

这层是公域流量池，帮助内容被看见、被传播、被验证。

他们最关心：

- 这是什么新东西
- 为什么大家都在聊
- 这事到底重不重要

## 5.2 第二层：科技从业者 / 准创业者

这是最关键的承上启下层。

他们最关心：

- 这条方向到底是不是机会
- 现在入场是太早、太晚还是刚刚好
- 谁在做、怎么做、哪里可能有空档

## 5.3 第三层：创业者 / 创始团队 / 产业方

这是高价值目标对象。

他们最关心：

- 你是不是有行业 sense
- 你是不是只追热点，还是能形成有价值的观察
- 你是不是值得进一步交流

### 关键纪律

公域内容虽然能吸引更大众的人群，但表达时要始终兼顾：

> **让泛人群看得懂，让目标客户看得出你不是泛资讯号。**

## 6. 优先使用的内容模型

冷启动阶段优先用以下 4 种内容模型。

## 6.1 热点快拆

适用于：

- 新发布
- 新融资
- 新产品
- 新政策
- 新争议

标准结构：

1. 先说结论
2. 发生了什么
3. 为什么重要
4. 它影响谁
5. 下一步最该看什么

## 6.2 机会清单

适用于：

- 本周值得关注的方向
- 某个赛道的关键变化列表
- 值得跟踪的公司 / 产品 / 趋势

标准结构：

1. 给出清单
2. 每个点一句核心看点
3. 标出哪些值得继续深看

## 6.3 误区纠偏

适用于：

- 某个热点被过度解读
- 大家都在说，但理解跑偏

标准结构：

1. 主流说法是什么
2. 问题在哪里
3. 更合理的看法是什么
4. 真正该盯的变量是什么

## 6.4 赛道入门解释

适用于：

- 新技术开始热起来
- 某个方向门槛高、公众看不懂

标准结构：

1. 这个方向是做什么的
2. 为什么最近又被关注
3. 关键角色 / 技术 / 应用场景
4. 当前该怎么看，不该怎么看

---

## 7. 公域内容必须具备的表达特征

## 7.1 信息密度高，但读起来不费劲

要做到：

- 少说空话
- 少堆黑话
- 每一段都推进认知

## 7.2 有判断，但不装全知

允许：

- 给出观察
- 给出优先级
- 给出“接下来该看什么”

不允许：

- 把推测写成事实
- 把概率判断写成必然结果
- 没有依据时强做结论

## 7.3 像“懂行的人在解释”，不是“卖课的人在喊单”

要克制、清楚、有人味。

不做：

- 夸张煽动
- 装懂式大词
- 强贩卖感

## 7.4 先解释，再引导

转化钩子必须建立在内容本身有价值的前提上。

原则是：

> 先让用户觉得你讲得清楚，再让他愿意继续靠近。

## 8. 推荐语气

### 8.1 推荐气质

- 专业
- 清楚
- 克制
- 有判断
- 不端着
- 不油腻

### 8.2 推荐句式

可多用下面这类表达：

- 这件事值得看，不在于表面热闹，而在于……
- 真正要看的不是结论本身，而是……
- 如果把它放回产业链里看，重点在……
- 先别急着下结论，至少还有三个变量要继续看……
- 对普通读者来说，先理解这一步就够了……

### 8.3 避免句式

- 颠覆一切
- 普通人最后上车机会
- 这将彻底改变世界
- 错过再等十年
- 全网没人讲清楚，只有我们看懂了
- 100% 会怎样

## 9. 公域内容边界

## 8.1 可以讲的

- 公开事件
- 公开公司动态
- 公开产品发布
- 公开舆论热点
- 公开趋势信号
- 公开资料基础上的解释和判断

## 8.2 不要讲透的

- 依赖闭门访谈才能成立的核心判断
- 不适合公开扩散的敏感交流内容
- 仍在形成中的深度认知底稿
- 会明显暴露私域价值的核心交付物

## 8.3 明确不能讲的

- 未经验证的爆料
- 暗示掌握内幕
- 拿不出证据的强结论
- 容易引发法律或平台风险的说法

---

## 9. 公域内容发布前自检

发布前至少问 7 个问题：

1. 这条内容的核心价值是筛选、解释、机会感还是纠偏？
2. 用户能不能在 10 秒内看懂它在讲什么？
3. 这条内容有没有一句明确的“为什么值得看”？
4. 它是单纯复述新闻，还是有增量？
5. 用词是否克制，是否存在夸张和硬拗结论？
6. 是否照顾到了目标客户的识别感？
7. 是否留下了一个自然、不硬推销的后续交流钩子？

---

## 10. 一句话纪律

同行资本的公域内容，不追求一上来就把最深的认知全部讲完，而是要稳定做到：

> **筛选重要变化，解释关键逻辑，提示机会感，并让对的人愿意继续靠近。**
