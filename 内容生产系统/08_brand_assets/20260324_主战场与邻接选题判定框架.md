# 同行资本市场内容系统｜主战场与邻接选题判定框架

## 1. 这份文档解决什么问题

这份文档用来解决一个很关键的问题：

> 同行资本的公域内容，不能又散又泛，但也不能把自己锁死到过窄的话题里。

所以我们不采用“70% 做什么、30% 做什么”这种死比例，而采用：

> **主战场明确 + 邻接扩展灵活 + 漂移时自动纠偏**

这是一个更适合 skill 化执行的方法。

---

## 2. 内容范围先钉死：只做 AI 圈

同行资本公域内容，第一层边界先钉死：

> **只做 AI 圈相关内容。**

不碰泛科技大杂烩，不为了追热点去讲与 AI 无关的话题。

但 AI 圈内部，不是只有 agent 本身可以讲。

我们采用“主战场 + 邻接领域”的方法。

这里有一个非常重要的前提：

> **主战场 / 邻接领域，是选题判断框架，不是信息源等级制度。**

也就是说：

- 信息源层面，不应该因为“更偏主战场”就天然高人一等
- 只要是 AI 圈里真实有热度、真实有讨论、真实容易形成好选题的源，都应该被公平纳入
- 真正的取舍，应该发生在 **候选题推荐阶段**，而不是发生在 **信息输入阶段**

---

## 3. 主战场是什么

同行资本的**主战场**，不是笼统的“AI 新闻”，而是下面这几类内容：

## 3.1 Agent 本体

- agent 产品
- agent workflow
- agent orchestration
- agent 自动执行能力
- agent 在真实业务中的效果与限制

## 3.2 一人公司 / 超级个体 / agent 自主运营业务

- 一人公司案例
- 用 agent 跑业务的真实实践
- agent 如何替代岗位或串起工作流
- 哪些业务适合 agent-first 地做

## 3.3 好的 skill / workflow / stack

- 值得配置的 skill
- 值得复用的工作流
- 值得关注的工具链组合
- 哪些 stack 正在成为新的行业共识

## 3.4 真实结果导向的案例拆解

- 为什么这个业务能快速拿到结果
- 为什么这个 AI 项目突然爆火
- 它的底层增长逻辑、供给逻辑、分发逻辑是什么
- 我们还能从中抄到什么方法

## 3.5 同行资本自己的 build in public

- 我们这周在做什么
- 哪些 agent 调整有效
- 哪些 bug 被修掉了
- 哪些自动化真正带来了结果

一句话总结：

> **主战场就是：agent 能力、agent 业务、agent 工具链、agent 自主运营案例。**

---

## 4. 哪些内容可以作为邻接领域

主战场之外，下面这些内容也可以讲，但前提是：

> **它必须与主战场有明确因果关系。**

## 4.1 模型重大更新

模型更新不是因为“大公司发新模型”就要讲，而是因为它可能带来：

- agent 可靠性提升
- tool use / browser use / coding / planning 能力变化
- 成本变化
- latency 变化
- multimodal 能力变化
- 长时任务可行性变化

### 能讲的角度

- 这个模型更新，会不会让某类 agent 工作流从“玩具”变“可用”
- 会不会让一人公司某类业务 ROI 发生变化
- 会不会改变现有 agent stack 的最佳实践

## 4.2 Agent Infra 的重大变化

例如：

- memory
- evals
- orchestration
- browser/computer use
- voice stack
- deployment
- observability
- safety / permissions

这类内容很重要，因为它们直接影响：

- agent 是否能稳定跑起来
- agent 是否能进真实业务
- 一人业务是否能真正自动化

## 4.3 AI 硬件 / AI 终端

不是所有 AI 硬件都要讲，只有当它具备下面一种以上价值时才讲：

1. agent-native 交互明显更强
2. 带来了新的 agent 使用场景
3. 有明显卖点和传播性
4. 能引申出“AI 终端如何承接 agent”的重要讨论

## 4.4 平台、分发、政策、生态事件

例如：

- 新的平台接口开放
- 某大平台开始支持 agent 分发
- 某政策明显利好 agent 创业 / 一人公司
- 某类收费、支付、权限体系变化

如果这些变化会直接影响 agent 业务能否落地，就值得讲。

---

## 5. 什么内容不该讲

下面这些内容，即便属于 AI，也不应该自动进入候选池：

1. 纯模型跑分新闻
2. 与 agent 或业务落地没有关系的泛模型八卦
3. 纯资本市场热闹、但没有产品和业务含义的新闻
4. 纯概念炒作，没有 workflow、产品、业务映射的内容
5. 看起来很前沿，但和“agent 如何创造结果”完全无关的话题

换句话说：

> **不是“AI 就能讲”，而是“这件 AI 变化能不能回到 agent 主战场”。**

---

## 6. 实操时怎么判断一个题该不该做

我们不用死比例，改用 4 步判定法。

## 6.1 第一步：是不是 AI 圈内容

不是 AI，直接不做。

## 6.2 第二步：是不是主战场内容

只要直接命中下面任意一类，就优先级天然更高：

- agent 产品 / 能力 / workflow
- 一人公司 / 超级个体 / agent 自主运营案例
- skill / workflow / stack 拆解
- 同行资本自己的真实进展

如果是，直接进入主候选池。

## 6.3 第三步：如果不是主战场，它能否反向改变主战场

问三个问题：

1. 它会不会显著改变 agent 的能力边界？
2. 它会不会显著改变 agent 业务的成本、效率或可行性？
3. 它会不会显著改变 agent 的交互、分发或终端形态？

只要至少有一个答案是“会，而且很明显”，它就可以作为**邻接高价值题**进入候选池。

## 6.4 第四步：能不能把题目拉回主战场

如果最后生成标题和大纲时，没办法自然回答下面这句话：

> **这件事对 agent、对一人公司、对真实业务落地到底意味着什么？**

那就说明它不该做，或者至少不该现在做。

---

## 6A. 信息源中立，筛选后移

为了避免系统因为“预设立场”而错过大热点，我们明确区分两层：

## 6A.1 信息源层：尽量公平、尽量完整

信息源层的任务只有一个：

> **把 AI 圈里值得看的变化尽可能完整地抓回来。**

这一层不做下面这些事：

- 不因为题材“不在主战场”就压低源的等级
- 不因为某个源更偏模型 / 硬件 / 终端，就先天边缘化
- 不因为某个源暂时和 agent 关系不强，就放弃监控

这一层真正该看的，是：

- 热度
- 增速
- 讨论度
- 可写性
- 信号质量
- 是否持续产出好题

## 6A.2 候选题层：统一评分，后道筛选

真正的品牌取舍，应该放到候选题层来做。

更合理的方法是：

1. 先用一套相对公平的标准，筛出一批高潜选题
2. 再基于同行资本的主身份，做第二轮优先级调整

例如：

- 先从当天所有 AI 热点里，筛出 8 个强候选
- 其中 2 个虽然很热，但和主战场关联偏弱
- 那么在推荐给创始人时，可以保留这 2 个的记录，但优先建议另外 6 个
- 如果那 6 个里有 3 个同时又更能强化我们的主身份，就把这 3 个顶到最前面

这叫：

> **不牺牲市场感觉，但在最后推荐时体现品牌纪律。**

---

## 6B. 候选题推荐时的两段式逻辑

## 6B.1 第一段：市场视角先筛

先看：

- 热度是不是足够高
- 讨论是不是正在扩散
- 是否容易形成清晰标题
- 是否容易讲明白
- 是否具备传播性或参与感

## 6B.2 第二段：品牌视角再排

再看：

- 是否属于主战场
- 如果不属于主战场，是否能强力反哺主战场
- 发这条是否会强化同行资本的核心身份
- 如果连续发类似内容，会不会让账号逐渐跑偏

这两个阶段缺一不可。

如果只有第一段，账号会越来越泛；
如果只有第二段，账号会越来越自嗨。

我们要的是：

> **先尊重市场，再体现身份。**

---

## 7. 不设死比例，怎么防止内容跑偏

不设死比例，不等于没有纪律。

我们用“漂移纠偏”来代替固定配比。

## 7.1 漂移的定义

如果最近连续几条内容都出现下面情况，就说明开始跑偏了：

- 都是泛 AI 热点
- 都在讲模型大新闻
- 都没有回到 agent / 一人业务 / workflow
- 看起来热闹，但和同行资本的核心特色越来越弱

## 7.2 纠偏规则

一旦出现跑偏迹象，下一批候选题优先补下面几类：

1. agent 实战案例
2. 一人公司业务拆解
3. skill / workflow 拆解
4. 同行资本自身进展
5. 从当天高热题里，优先挑那些能自然回扣主战场的题

也就是说：

> **不靠比例约束，而靠“最近内容是否还在强化我们的主身份”来纠偏。**

---

## 8. 最推荐的内容结构

为了兼顾灵活性和聚焦度，我建议把所有内容放入 3 个层级。

## 8.1 A 类：主战场核心内容

这是最能代表同行资本身份的内容：

- agent 工作流
- 一人公司案例
- agent 业务拆解
- skill / stack 拆解
- TH build in public

## 8.2 B 类：主战场增强内容

这些内容不直接讲 agent 本体，但会强力影响 agent：

- 模型升级
- agent infra
- 权限 / deployment / observability
- 支付 / 平台 / 分发变化

## 8.3 C 类：主战场外缘内容

这些内容可以讲，但门槛最高：

- AI 终端
- AI 硬件
- 大型 AI 事件

只有当它们真的有明显卖点，且能清楚回扣到 agent 业务、agent 交互或 agent 创业时，才值得做。

---

## 9. skill 执行时的核心要求

后续让 agent 跑公域内容时，不应让它机械执行“比例规则”，而是要让它学会下面这套思考方式：

1. 这个题是不是 AI 圈
2. 这个题是主战场，还是邻接领域
3. 如果是邻接领域，它到底如何改变主战场
4. 这个题是否能自然落到 agent / 一人业务 / workflow / skill / build in public
5. 这个题做出来，是强化了同行资本的身份，还是稀释了它

---

## 10. 一句话结论

同行资本的公域内容，不是做“泛前沿科技号”，也不是把自己锁成“只讲 agent 小圈子”。

更准确的做法是：

> **以 agent、自主运营、一人公司、skill/workflow 为主战场；对模型、infra、硬件、平台等邻接 AI 热点，只有在它们会实质改变主战场时才展开。**
