# 同行资本市场内容系统｜平台定位与表达规范

## 1. 总原则

不同平台不是把同一篇稿子机械改写，而是承担不同任务。

所有平台统一内容边界：

- 只做 AI 圈
- 以 agent、自主运营、一人公司、skill/workflow 为主战场
- 对模型、infra、AI 硬件、平台变化，只在它们会明显改变主战场时展开

详细判定见：

- `/Users/apple/Documents/同行资本内容部门/内容生产系统/08_brand_assets/20260324_主战场与邻接选题判定框架.md`

本阶段的正式支持平台为 7 个：

- 微信公众号
- 小红书
- 知乎
- X
- B站专栏
- 今日头条
- 百家号

统一纪律：

1. 同一主题允许跨平台复用，但**必须重算表达**。
2. 不是每个题都要 7 平台齐发，要先定平台束，再写稿。
3. 任何平台都不得丢失：
   - 核心判断
   - 背景桥接
   - 关键引用依据
   - 风险提示

## 3. 微信公众号

### 3.1 平台角色

公众号是 **“讲清楚 + 留资产”** 的核心阵地。

### 3.2 适合的内容模型

- 热点快拆的完整版
- 趋势解释
- build in public 周报 / 复盘
- 中深度专题
- 重点案例拆解

### 3.3 推荐结构

1. 标题先讲清主题
2. 开头 2-4 段快速说清“为什么值得看”
3. 中段补背景与变量
4. 中后段给 TH Capital 判断
5. 结尾落到下一步观察点 + 轻 CTA

### 3.4 标题原则

- 一眼知道在讲什么
- 有现实关切
- 有“值得继续读”的理由

### 3.5 语气要求

- 专业但不端着
- 有结构但不学术腔
- 有判断但不过满

### 3.6 禁忌

- 像资讯搬运
- 像研究报告原文
- 大段空话和套话

## 4. 小红书

### 4.1 平台角色

小红书承担的是 **“把复杂内容翻译成低门槛入口”**。

### 4.2 适合的内容模型

- 热点速懂
- 一图看懂
- 误区纠偏
- 机会清单
- 低门槛入门解释

### 4.3 推荐结构

1. 先抓注意力
2. 快速给结论
3. 用 3-5 个点讲清楚
4. 至少留一个“可收藏信息块”
5. 结尾做轻互动 / 轻导流

### 4.4 语气要求

- 像懂行的人在帮用户翻译
- 不写成研报腔
- 不居高临下

### 4.5 禁忌

- 大段长文
- 黑话堆砌
- “我比你懂很多”的说教口气

---

## 5. 知乎

### 5.1 平台角色

知乎承担 **“问题导向的解释与认知承接”**。

### 5.2 适合的内容模型

- 回答型内容
- 技术路线 / 商业路线解释
- 争议话题的结构化回应
- evergreen 的概念澄清文

### 5.3 推荐结构

1. 先直接回答问题
2. 再补背景
3. 再讲逻辑和分歧点
4. 最后给当前阶段判断

### 5.4 语气要求

- 比公众号更直接
- 比小红书更系统
- 要有“我真的回答了问题”的感觉

### 5.5 禁忌

- 只给观点不论证
- 开头像导语，不像回答
- 过度情绪化表达

## 6. X

### 6.1 平台角色

X 更像一个 **外部信号面板 + 海外快评窗口**。

### 6.2 适合内容

- 单条快评
- thread
- 关键链接汇总
- 海外观点串讲

### 6.3 推荐结构

单条：

1. 一句话判断
2. 一句背景桥接
3. 一句 why-you-should-care

thread：

1. 先给总结
2. 分条拆
3. 最后一条给 follow-up angle

### 6.4 语气要求

- 短
- 准
- 每条都要有信息增量

### 6.5 禁忌

- 中文长文直接压缩
- 连续几条都在重复一个意思
- 只喊情绪，不给判断

## 7. B站专栏

### 7.1 平台角色

B站专栏承担 **“builder 社区深度补位”**。

它更适合：

- 案例拆解
- workflow / skill / stack 讲解
- 工具实操复盘
- 一人公司 / agent 运营过程拆解

### 7.2 推荐结构

1. 标题直接点题，不卖关子
2. 开头先说“这篇能帮你解决什么问题”
3. 中段按步骤 / 模块拆解
4. 穿插具体例子、卡点、适用边界
5. 结尾给可执行建议或下一步路径

### 7.3 语气要求

- 像一个真正在干活的人复盘
- 允许更实操、更口语
- 比公众号更贴近“教程 / 经验分享”

### 7.4 禁忌

- 空泛概念文
- 只有判断，没有过程
- 过度 SEO 化标题

## 8. 今日头条

### 8.1 平台角色

今日头条承担 **“大众表达测试 + 标题包装实验”**。

### 8.2 适合的内容模型

- 热点快拆
- 为什么值得看
- 3 点看懂
- 清单型 / 误区型 / 口语型内容

### 8.3 推荐结构

1. 标题先给冲突 / 结果 / 反常识
2. 开头快速补一句背景
3. 正文用 3-5 个短块讲清楚
4. 结尾给一个明确判断

### 8.4 语气要求

- 更口语
- 更直接
- 允许一定程度的大众表达，但不能低俗

### 8.5 禁忌

- 为了流量把判断写歪
- 只剩标题党，没有信息增量
- 完全照搬公众号结构

## 9. 百家号

### 9.1 平台角色

百家号承担 **“百度搜索镜像与长尾承接”**。

### 9.2 适合的内容模型

- 什么是……
- 为什么……
- 如何看待……
- 路线对比 / 框架解释 / 问题整理

### 9.3 推荐结构

1. 标题围绕问题或关键词
2. 开头 1 段先给结论
3. 中段按定义 / 背景 / 变量 / 风险展开
4. 适度重复关键名词，但不生硬堆词
5. 结尾回到当前判断与观察点

### 9.4 语气要求

- 清楚
- 稳定
- 可检索

### 9.5 禁忌

- 只为 SEO 堆关键词
- 写成没有人味的百科稿
- 完全脱离当前事件语境

## 10. 跨平台改写纪律

### 10.1 不做简单裁切

同一个主题，平台版本必须重算：

- 用户状态
- 打开场景
- 最适合的切入角度
- CTA 类型

### 10.2 先定目标，再定表达

每个平台出稿前先回答：

1. 这条内容是拉新、建立信任，还是承接搜索？
2. 这平台用户现在最需要哪种信息密度？
3. CTA 是关注、收藏、评论、私信，还是不加 CTA？

### 10.3 同一主题允许信息深浅不同

允许：

- 小红书只讲最关键的 3 个点
- 微信讲 5-7 个点
- 知乎讲清争议和逻辑
- X 只讲一个判断和一个链接
- B站专栏讲过程与卡点
- 头条讲大众表达版本
- 百家号讲搜索友好版本

这不是信息不一致，而是分层表达。

## 11. 一句话总结

平台策略的核心不是“多发”，而是：

> **让每个平台都承担最适合自己的增长和认知任务。**
