## 4. 吸收的 4 种落地方式

后续每一项外部能力，只允许以下 4 种落地方式之一。

### 4.1 落到内部 skill

适用于：

- 方法论
- 平台写法
- 决策规则
- 风险边界
- 复盘判断逻辑

典型例子：

- 把 `humanizer` 的核心逻辑吸收到 `th-content-polish`
- 把 `wechat / xiaohongshu / twitter` 的平台逻辑吸收到 `th-market-platform-renderer`

### 4.2 落到 runbook / workflow

适用于：

- 任务顺序
- 人工确认点
- 多 agent 交接点
- 异常重试与兜底

典型例子：

- 把 `multi-agent-setup` 吸收到内容工厂的角色拆分与交接规则
- 把 `n8n-automation` 的重试 / human-in-the-loop 思想吸收到 runbook

### 4.3 落到脚本 / 工具层

适用于：

- 反爬抓取
- 网页净化
- 结构化 builder
- 文档导出
- 封面 / 配图 / SEO 辅助

典型例子：

- 把 `web-scraper` / `distil-web` 的能力吸收到 source capture 和 builder
- 把 `video-summary` 的能力吸收到视频来源转文字链

### 4.4 落到 agent 分工与前台协同

适用于：

- 谁对老板说话
- 谁负责补料
- 谁负责平台化包装
- 谁负责复盘与改进建议

典型例子：

- `market-editor` 继续做唯一前台总编
- `market-scout` 继续做后台信号官
- 后续再按瓶颈决定是否启用 `topic-planner / content-writer / publish-ops / content-analyst`

## 5. 施工总原则

### 5.1 A 档必须进入主施工表

A 档不是“以后有空再吸收”，而是这轮就要进入我们的正式施工队列。

### 5.2 B 档必须理解吸收，但不许把主链做重

B 档要做两件事：

1. 现在就预留接口和规则
2. 真正接入时只接高价值部分

不能因为 B 档很诱人，就把主链改成复杂大工程。

### 5.3 外部 skill 只能提供“思路与机制”，不能接管业务

严禁：

- 直接照抄 prompt
- 直接照搬角色分工
- 直接整包安装当成能力完成
- 让外部 skill 的默认打法替代我们的平台战略

### 5.4 本轮仍坚持一个硬边界

> **内容工厂继续独立运行，不与研究线 agent 混用。**

### 8.4 这一步的明确施工任务

#### P0-1｜把搜索升级成“多引擎 + 原始来源优先 + 多语言”

必须吸收：

- 不是搜到一条就算完成
- 优先找官方源、原始帖、创始人原话、原始产品页
- 海外题默认中英双语补检

落地结果：

- `source packet` 要显式标记 `source_type / primary_source / language / verification_status`

#### P0-2｜把抓取升级成“反爬 + 失败兜底 + 净化提取”

必须吸收：

- 正常方式优先
- 失败后自动切到备用抓法
- 抓回来不是直接扔给写稿，而是先净化、去噪、保留源链接

落地结果：

- `source packet` 同时保留 `raw_capture` 与 `distilled_body`

#### P0-3｜把热点入口升级成“热榜输入，不直接决定选题”

必须吸收：

- 热榜是雷达，不是编辑部总编
- HN、热榜、Reddit、竞品内容都只是候选输入源
- 进入 Top 8 / Top 5 前必须再做证据补查

落地结果：

- `topic candidate` 需要显式写 `heat_signal` 与 `evidence_signal` 两套分数

#### P0-4｜把竞品研究升级成“三位一体”

必须吸收：

- 研究的不只是内容竞品
- 还要研究产品竞品和话题竞品
- 回答“为什么这个题该我们写”

落地结果：

- Radar brief 必须出现 `竞品在说什么 / 我们能不能说得更好 / 我们差异化切口是什么`

### 8.5 这一步的验收标准

达到以下 5 条，才算 Phase 1 完成：

1. 每个进入 Top 8 的话题，都有明确原始来源或明确单源警告
2. 每个 source packet 都能区分“原始抓取文本”和“净化正文”
3. 海外题默认能做一轮双语补查
4. 热榜不会直接把垃圾热点推成高优先级选题
5. Top 8 / Top 5 的证据链明显比现在更完整

### 9.4 这一步的明确施工任务

#### P0-5｜把“背景理解”前置，不再默认读者都懂

必须吸收：

- 每篇稿都要解决“这到底是什么事”
- 背景不一定死板放开头，但必须在用户掉线前补齐
- 背景、钩子、观点之间要有承接关系

落地结果：

- 每个成稿必须带 `context bridge`

#### P0-6｜把标题 / 封面 / 首屏当成一体生成

必须吸收：

- 不是单独起标题
- 而是“点开前承诺”和“点开后兑现”一体设计
- 不允许封面很猛、正文很散

落地结果：

- 每个平台版本至少包含：
  - 标题候选
  - 首屏 opening
  - 封面文案或封面信息摘要
  - 为什么这一组更容易点击

#### P0-7｜把平台专属写法写进 renderer，不再靠泛改写

必须吸收：

- 公众号：结构完整、信任感、长文节奏
- 小红书：轻门槛、抓钩子、信息块短平快
- X：thread 节奏、观点切片、即时感
- 知乎 / B站 / 头条 / 百家号仍按我们既有矩阵继续保持差异

落地结果：

- renderer 层要从“换个格式”升级成“换个平台表达逻辑”

#### P0-8｜把去 AI 味升级成“人格一致性”

必须吸收：

- 去 AI 味不是口水话
- 不只是口语化，而是更像“同行资本在说”
- 既要自然，也要保留信息密度

落地结果：

- polish 阶段必须检查：
  - 是否像真人在说
  - 是否像同行资本在说
  - 是否还保留关键事实与引用

### 9.5 这一步的验收标准

达到以下 5 条，才算 Phase 2 完成：

1. 所有平台稿都具备明确背景承接
2. 标题 / 首屏 / 主张之间不再脱节
3. 平台版本之间不再只是机械换皮
4. 去 AI 味后，核心信息没有被冲淡
5. 终稿比当前版本明显更像“可真实发布”的成品

## 10. 当前这一轮真正该从哪里开工

如果现在就开始正式施工，最合理的开工点只有一个：

> **先改 Phase 1，也就是先做 A 档情报底盘升级。**

理由很简单：

1. 现在最容易把系统带偏的，是“信号质量不稳”
2. 没有稳定证据链，后面的包装优化会越做越像化妆
3. 一旦情报底盘升级完成，后面的标题、背景、平台化写法都会更有抓手

所以本轮真正的第一铲土，就是：

- 先改 `th-seed-refresh-and-source-scouting`
- 再改 `th-source-capture-and-citation`
- 再改 `th-topic-radar`
- 然后联动调整 capture / radar 相关 runbook 与 builder
