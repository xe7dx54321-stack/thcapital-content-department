# 同行资本市场内容系统｜前台同步与展示层方案

## 1. 这份文档解决什么问题

内容工厂前面已经把：

- `热点抓取`
- `Top 8 -> Top 5`
- `approved_topic`
- `Draft Pack`
- `publish queue / review`

这条业务主链基本搭起来了。

但如果只有底层对象落盘，没有前台展示层，那么老板在飞书群里仍然会遇到两个问题：

1. 看不清内容工厂今天到底在做什么
2. 看不清当前到底卡在哪个阶段、需不需要人类介入

所以这一层要补的是：

> **让 `market-editor` 成为内容工厂唯一前台 bot，并且持续把关键节点、阶段结果、待人工动作与风险点，稳定同步到前台群。**

---

## 2. 目标

本轮前台同步层只做三件事：

1. **做一张每日状态板**
   - 让老板一眼知道内容工厂今天的正式任务、实际工作进度、阶段成果、后续计划、人工协助点。

2. **做关键节点同步**
   - 不是每个底层动作都发群，而是在真正影响业务推进的节点才同步。

3. **做最小展示层资产**
   - 先用 Markdown 状态板 + 飞书群简报实现“可见性”。
   - 后续控制台 / 看板再来消费这层结构化资产。

---

## 3. 前台同步的硬规则

### 3.1 只同步关键节点，不刷屏

前台群只同步这几类事件：

1. 今日 `Top 8 -> Top 5` 建议单完成
2. 某个题被正式拍板，已进入 `approved_topic`
3. 某个 `Draft Pack` 已进入 `ready / waiting_human_publish`
4. 出现明确 blocker
5. 有待人工动作
6. 晚盘收口

### 3.2 每次同步都回答同一组问题

每次对前台群同步，都尽量回答下面 5 个问题：

1. 当前有什么正式任务？
2. 当前实际在做哪个题 / 哪个阶段？
3. 当前阶段性成果是什么？
4. 下一步要做什么？
5. 是否需要人类协助？

### 3.3 全量信息进状态板，群里只发压缩版

- 群里只发短摘要
- 详细信息统一写进：
  - `11_frontstage/YYYYMMDD__market-frontstage-board.md`

### 3.4 展示层不替代业务对象

前台状态板只是展示层，不是新的业务对象主档。

正式对象仍然以这些目录为准：

- `02_topic_radar/`
- `03_topic_candidates/`
- `04_approved_topics/`
- `05_draft_packs/`
- `06_publish_queue/`
- `07_performance_reviews/`

## 4. 展示层新增资产

### 4.1 每日状态板

输出目录：

- `/Users/apple/Documents/同行资本内容部门/内容生产系统/11_frontstage/`

标准文件：

- `YYYYMMDD__market-frontstage-board.md`

作用：

- 给老板看今日全局状态
- 给 `market-editor` 做群同步时引用
- 给后续控制台 / 仪表盘消费

### 4.2 群同步草稿

不单独建对象文件，先写在每日状态板内：

- `## 群同步草稿`

这样做的目的是：

- 保持“状态板”和“前台发言”一致
- 降低 `market-editor` 自己总结时跑偏的概率

---

## 5. 每日状态板最少要包含什么

建议固定包含：

1. `Snapshot`
   - 今日 source packets / asset chains / topic clusters / approved topics / draft packs / waiting_human_publish / published

2. `当前正式任务`
   - 当前内容工厂还有哪些正式动作没闭环

3. `当前实际在做`
   - 当前最核心的推进焦点，不要泛泛而谈

4. `今日阶段性成果`
   - 今天已经做出了什么

5. `当前活跃对象池`
   - 哪些 topic 正在推进，推进到哪一步

6. `下一阶段计划`
   - 接下来要做什么

7. `人类协助`
   - 待拍板 / 待发布 / 待回填数据 / 待补资源

8. `今日日志时间线`
   - 让人快速回看今天关键动作

9. `群同步草稿`
   - 直接可发飞书群的压缩版同步文案

---

## 6. 关键节点与展示层的关系

### 6.1 Topic Radar 完成时

前台重点同步：

- 今日是否已经形成正式建议单
- 是否供给不足
- 哪 1-2 个题最值得先拍板

### 6.2 Approved Topic 形成时

前台重点同步：

- 老板已经选了哪个题
- 归一后的角度是什么
- 接下来会起哪些平台的稿

### 6.3 Draft Pack / Polish 完成时

前台重点同步：

- 哪个题已经进入可编辑稿状态
- 哪些平台版本已经齐
- 当前是 `drafting / ready / waiting_human_publish`

### 6.4 Publish Queue 阶段

前台重点同步：

- 哪些平台已等人工发
- 哪些平台已发
- 是否已经进入 review
