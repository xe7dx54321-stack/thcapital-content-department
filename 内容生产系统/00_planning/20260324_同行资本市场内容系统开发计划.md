# 同行资本市场内容系统开发计划（需求收敛版）

## 1. 项目定位

本项目独立于`/Users/apple/Documents/同行资本vc部门/VC系统开发规划`，单独作为同行资本对外内容输出系统建设，不与当前虚拟 VC 内部研究/项目推进体系混放。

这套系统要解决的不是“自动发几篇文章”，而是为同行资本建立一套**持续发现选题、提炼观点、生成多平台内容、沉淀发布资产、逐步形成市场影响力**的内容运营机制。

建议将这个新 agent 的定位收敛为：

> **市场部专员 / 内容市场 Agent**  
> 一个围绕“热点雷达 → 选题建议 → 人工确认 → 多平台稿件 → 发布提醒 → 复盘学习”运转的内容生产系统。

---

## 2. 业务目标

### 2.1 核心目标

1. 持续监控同行资本相关赛道的热点与高表现内容
2. 每天给创始人输出**高质量 Top 5 选题建议**
3. 在人类确认题目后，快速产出**适配不同平台格式的成稿/准成稿**
4. 持续积累同行资本的内容资产、写作风格、品牌表达与高表现选题经验
5. 最终形成一个“越用越懂内容”的市场内容系统

### 2.2 北极星指标（建议）

系统早期先不看虚荣指标，先看效率与命中率：

- 选题建议采纳率
- 采纳后成稿交付时长
- 成稿可直接发布率
- 单篇内容的跨平台复用率
- 被创始人判定为“有信息量 / 值得发”的比例

系统中后期再逐步纳入：

- 阅读/曝光/点赞/收藏/转发/评论
- 新增关注与品牌搜索提升
- 来自内容的潜在项目线索 / 行业连接 / 商务线索

## 3. 需求收敛后的核心范围

## 3.1 这次先要做什么

建议把第一阶段需求收敛为 5 件事：

1. **跨平台热点监控**  
   监控公众号、小红书、X、知乎、B站、抖音、TikTok、YouTube 等平台中，和同行资本定位相关的热点内容、头部账号、竞品账号、关键人物。

2. **Top 5 选题建议**  
   每天输出一份市场部选题建议，给出：
   - 建议标题
   - 为什么值得做
   - 在哪些平台发酵
   - 原始链接
   - 建议切入角度
   - 适合哪些平台
   - 风险提示

3. **人工确认机制**  
   创始人只需要回复“选第 3 个”或者“第 2 个改成机构视角”，系统再继续往下推进。

4. **多平台稿件生成**  
   根据确认选题，自动生成一套 Draft Pack，例如：
   - 微信公众号版本
   - 小红书版本
   - 知乎版本
   - X / Twitter 线程版
   - B站专栏版
   - 今日头条版
   - 百家号版

5. **发布提醒与后续复盘**  
   先做到“稿件入库 + 等待发布 + 提醒人类去发”，后续再逐步对接草稿箱与发布后效果复盘。

## 3.2 这次先不做什么

为了避免一开始就做成一个重而杂的系统，建议明确非目标：

1. **先不追求全自动发布**  
   平台账号安全、登录态、验证码、风控、草稿箱接口差异很大，第一阶段不应该把主要精力耗在这里。

2. **先不做“完全无人审批”的自动发文**  
   对外内容直接代表品牌，不适合在需求尚未稳定时完全放权。

3. **先不做“每题全平台强推”**  
   第一阶段虽然正式支持 7 平台，但依然按题目 fit 选择平台束，不做机械群发。

4. **先不做内容增长黑盒**  
   不是一开始就搞复杂推荐算法，而是先建立正确的工作流与结构化资产沉淀。

## 4. 推荐的系统形态

与其把它理解为“市场部机器人”，不如把它定义成 4 个能力模块：

1. **Topic Radar（选题雷达）**
2. **Topic Planner（选题策划）**
3. **Draft Studio（多平台写稿）**
4. **Publish Ops（发布提醒 / 后续复盘）**

早期实现上，不一定要拆成 4 个独立 agent，可以先由一个主 agent 承担 4 种模式；等工作流稳定后再拆。

### 4.1 推荐角色命名

建议后续系统中统一叫：

- `market-editor`：市场部主执行 agent
- 后续如需拆分，可扩展为：
  - `market-scout`
  - `topic-planner`
  - `content-writer`
  - `publish-ops`
  - `content-analyst`

## 5. 日常工作流（建议定版）

## 5.1 每日工作主链路

### A. 热点发现

系统从多平台抓取/观察：

- 竞品账号的新内容
- 头部 KOL / 媒体 / 公司创始人的新内容
- 平台趋势榜 / 热门榜 / 增长异常内容
- 与 AI、半导体、量子、具身智能、产业投资、科技创业、技术趋势有关的话题

### B. 主题归并

对采集到的素材进行聚类，不是逐条扔给创始人，而是归并成“话题簇”：

例如：

- “某个新模型发布”
- “AI Agent 商业化的新拐点”
- “某个技术方向在海外持续升温”
- “某个行业出现资本与订单共振”

### C. 选题评估

系统对每个话题簇打分，判断：

- 值不值得写
- 为什么现在写
- 适合用什么角度写
- 哪个平台更适合先发

### D. 输出 Top 5 建议

给创始人一份结构化建议单。

### E. 人工确认

创始人只需要做轻量决策：

- 选哪一个
- 是否修改角度
- 是否指定平台优先级

### F. 生成 Draft Pack

系统针对不同平台自动生成稿件，并同时输出：

- 标题候选
- 正文
- 配图建议 / 结构建议
- 发布备注
- 风险点

### G. 发布提醒

先不追求全自动发文，先做到：

- 草稿入库
- 待发布清单
- 飞书提醒创始人/运营去发

### H. 效果复盘

发布后记录效果，反向校正：

- 哪类话题容易被采纳
- 哪个平台对哪类内容更友好
- 哪种写法表现更好
- 哪些来源更容易产出高质量选题

## 7. 选题评分模型（建议）

系统不应该只看“热不热”，而应综合判断“值不值得同行资本来写”。

建议至少看以下 8 个维度：

1. **相关性**  
   是否和同行资本的核心定位相关：AI、半导体、量子、具身、科技投资、产业趋势、创业公司、技术商业化。

2. **跨平台发酵度**  
   是否不仅在单平台热，而是在多个平台同时被讨论。

3. **增长速度**  
   是否在短时间内出现异常增长，例如 24 小时内播放/互动明显上升。

4. **持续性**  
   是一瞬间的八卦流量，还是未来几天/几周仍有讨论空间。

5. **可写性**  
   是否能写出信息量，是否能写出同行资本自己的判断，而不是复述新闻。

6. **差异化空间**  
   同样一个热点，同行资本能不能给出更专业的切入角度。

7. **品牌契合度**  
   是否符合同行资本想建立的品牌形象：懂科技、懂产业、懂投资、有判断。

8. **合规与风险**  
   是否存在版权风险、误导性风险、过强的舆情争议风险。

### 7.1 建议的评分输出格式

每个候选题至少要输出：

- 选题名称
- 总体评级（S/A/B/C）
- 建议动作（立刻写 / 观察 / 不写）
- 核心理由
- 对应平台建议
- 原始链接列表

## 8. Top 5 建议单的标准格式

建议每天交付给创始人的不是碎片消息，而是一份结构化建议单。

每个建议项至少包含：

1. **建议标题**
2. **一句话判断**（为什么值得做）
3. **热度证据**
4. **平台分布**
5. **建议切入角度**
6. **建议输出形式**（深度文 / 观点短帖 / 问答 / 长文 / 视频脚本）
7. **原始链接**
8. **风险提示**

### 8.1 示例结构

```markdown
Top 3｜为什么 AI Infra 的叙事正在从“训练”切向“推理”
- 判断：适合做，且适合微信 + 知乎双平台同时发
- 核心理由：GTC 后多个平台持续讨论，且有同行资本擅长的产业链视角
- 热度证据：X 上头部从业者连续讨论；YouTube 相关视频 24 小时增速高；公众号已有多篇跟进
- 建议角度：不复述大会内容，而是回答“哪些公司会真正受益”
- 原始链接：...
- 风险：容易写成大会总结，必须加投资/产业判断
```

## 9. 成稿输出对象定义

## 9.1 Draft Pack 的组成

建议系统最终输出的不是单篇稿，而是一整套 Draft Pack：

1. **微信稿**
2. **小红书稿**
3. **知乎稿**
4. **X 线程版**
5. **标题备选**
6. **摘要版 / 导语版**
7. **配图或图表建议**
8. **引用来源清单**
9. **发布建议备注**

## 9.2 平台稿件的差异化要求

### 微信公众号

- 长度：1500–3500 字为主
- 目标：形成品牌判断与体系化表达
- 结构：背景 → 观点 → 分析 → 结论 → 延伸
- 要求：不能像资讯搬运，必须有“同行资本为什么这么看”

### 小红书

- 长度：短平快、高信息密度
- 目标：更强的情绪抓手和观点抓手
- 结构：标题钩子 + 快速结论 + 3–5 个核心点 + 收尾
- 要求：能让人快速理解“为什么值得关注”

### 知乎

- 长度：中长文
- 目标：回答式、框架式、解释式
- 结构：先给结论，再给论证
- 要求：适合问题导向内容

### X / Twitter

- 长度：线程化、短句化
- 目标：快速表达核心判断
- 结构：Hook + 关键点列表 + one-line conclusion
- 要求：更适合跟热点、跟海外趋势

### B站专栏

- 长度：1200–3000 字为主
- 目标：做 builder 社区深度补位
- 结构：问题 → 方法 / 过程 → 卡点 → 结论
- 要求：更像真实复盘和教程，不要像泛资讯号

### 今日头条

- 长度：600–1800 字为主
- 目标：测标题、测大众表达、测泛流量反馈
- 结构：钩子标题 + 快速结论 + 3–5 个短块 + 收尾判断
- 要求：口语化，但不能丢判断

### 百家号

- 长度：1000–2500 字为主
- 目标：百度搜索镜像与关键词承接
- 结构：问题 / 关键词标题 → 结论 → 背景 → 变量 → 风险 → 总结
- 要求：可检索、可解释、不能写成纯百科堆词

### 抖音 / TikTok / YouTube（后续）

- 先输出脚本，不必第一阶段强做自动视频生产
- 可先包括：
  - 口播提纲
  - 分镜建议
  - 标题建议
  - 封面文案建议

## 10. 草稿箱与发布策略

这里非常关键：**“写入平台草稿箱”不能作为 MVP 的前置依赖。**

原因：

1. 各平台 API/开放能力差异大
2. 很多平台没有稳定草稿接口
3. 浏览器自动化存在登录态、验证码、风控、封禁风险
4. 一开始最重要的是把内容工作流跑顺，而不是卡在平台自动化上

### 10.1 推荐分阶段推进

#### Phase 1：统一 Draft Pack 工作台

先把稿件统一写入本地结构化目录 / 云文档空间：

- 一处查看
- 一处修改
- 一处确认待发布状态

#### Phase 2：半自动草稿箱接入

如果平台允许，则逐步接入：

- 浏览器自动化填入草稿
- 人工最后确认发布

#### Phase 3：按平台单独做正式发布适配

只针对真正高频、收益高的平台做更深集成。

## 12. 系统能力拆解

## 12.1 监控能力

需要具备的监控对象包括：

1. **竞品账号**
   - 同赛道投资机构账号
   - 科技媒体账号
   - 头部内容账号

2. **关键人物**
   - 创业者
   - 投资人
   - 科技媒体主编
   - AI / 半导体 / 量子 / 具身领域头部人物

3. **趋势入口**
   - 平台热榜
   - 搜索推荐词
   - 爆款增长榜
   - 高热视频/高传播帖文

4. **内部知识回流**
   - 同行资本内部研究结论
   - 已有行业图谱和历史判断
   - 历史高表现选题

## 12.2 分析能力

系统需要能回答：

- 这件事现在为什么热
- 是单点热点还是趋势拐点
- 同行资本写这件事的最佳角度是什么
- 适合深度讲还是快评讲
- 是否值得占用创始人发布时间窗口

## 12.3 写作能力

系统需要具备：

- 长文结构化表达能力
- 短内容钩子与切片能力
- 跨平台重写能力
- 品牌口径统一能力
- 引用与来源保留能力

## 12.4 复盘能力

系统后续需要积累：

- 什么题材容易被采纳
- 什么题材适合什么平台
- 什么写法更有效
- 哪些监控源贡献的好题更多

## 14. 建议的 Agent 工作机制

## 14.1 MVP 阶段

先用一个主 agent 跑通全流程：

- 早晨：生成 Topic Radar
- 中午前：输出 Top 5 选题建议
- 创始人确认后：产出 Draft Pack
- 晚上：记录发布与待发布状态

## 14.2 扩展阶段

后续再逐步拆分：

- `market-scout`：负责找热点
- `topic-planner`：负责选题与排序
- `content-writer`：负责写稿
- `publish-ops`：负责草稿/提醒/排期
- `content-analyst`：负责复盘

### 14.3 为什么不建议一开始就多 agent

因为这套业务流程当前还没有完全收敛，如果过早拆多 agent：

- 协调成本会很高
- 需求变化时改动面太大
- 很容易先把流程做复杂

所以建议：

> **先单 agent 跑通，再按瓶颈拆角色。**

## 15. 品牌与内容风格约束（后续必须补齐）

这部分现在还没有完全定义，但后续一定要建立成显式资产。

至少要补的内容有：

1. 同行资本想对外呈现什么品牌形象
2. 哪些内容可以写，哪些不该写
3. 是偏深度研究、偏行业判断、偏热点快评，还是三者结合
4. 语气是更专业克制，还是更锋利鲜明
5. 是否允许强结论、强观点、强预测
6. 是否允许引用未证实信息
7. 是否允许带项目观察、企业评价、投资倾向表达

建议后续建立：

- 品牌写作手册
- 平台风格手册
- 高频用语黑白名单
- 风险表达规则

## 16. 分阶段开发计划

## Phase 0：需求收敛与资产准备

### 目标

把市场内容系统的业务边界定清楚。

### 交付物

- 本开发计划文档
- 首版目录结构
- 首版平台优先级
- 首版内容工作流
- 待确认问题清单

### 完成标准

- 创始人认可工作流主链路
- 明确 P0 平台与 P1 平台
- 明确系统输出对象

---

## Phase 1：Topic Radar + Top 5 建议（MVP）

### 目标

先把“找什么、为什么值得写、怎么建议”跑顺。

### 核心能力

- 监控 watchlist
- 聚类热点
- 生成 Top 5 选题建议单
- 附原始链接与理由

### 交付物

- 每日 Topic Radar
- 每日 Top 5 建议清单
- 基础评分模型

### 完成标准

- 创始人每天能快速看懂建议
- Top 5 建议不再是碎片搬运
- 至少能稳定给出 1–2 个真正值得写的题

---

## Phase 2：人工确认 + Draft Pack 生成

### 目标

把“我选第几个”之后的动作完全衔接起来。

### 核心能力

- 接收人工确认
- 生成多平台稿件包
- 保留来源与引用
- 统一存档

### 交付物

- 平台稿件模板
- Draft Pack 标准输出结构
- 成稿与准成稿目录

### 完成标准

- 创始人选题后可快速拿到可编辑/可发布稿件
- 不同平台版本明显区分，而不是一稿多投硬贴

---

## Phase 3：发布队列 + 提醒机制

### 目标

把内容从“写完”推进到“待发可管”。

### 核心能力

- 发布队列管理
- 状态标记：待修改 / 待发布 / 已发布 / 暂缓
- 飞书提醒
- 发布检查清单

### 交付物

- 发布队列台账
- 发布提醒模板
- 最小可用发布 SOP

### 完成标准

- 稿件不再散落在各处
- 每篇内容处于什么状态一目了然

---

## Phase 4：草稿箱 / 平台适配集成

### 目标

在确保安全与稳定的前提下，逐步提高“从稿件到发布”的自动化程度。

### 核心能力

- 按平台评估自动化可行性
- 半自动填草稿
- 发布前人工最终确认

### 注意事项

- 这不是前期主战场
- 必须以账号安全为前提
- 只给高收益平台做重度集成

---

## Phase 5：效果复盘 + 经验回灌

### 目标

让系统越来越懂“同行资本发什么内容更有效”。

### 核心能力

- 记录内容表现
- 记录题材表现
- 记录平台适配效果
- 校正来源权重与选题权重

### 交付物

- 周报 / 月报复盘
- 高表现内容库
- 低表现内容的失败归因

---

## 17. 技术实现建议

## 17.1 第一原则

**先做工作流正确，再做自动化深度。**

## 17.2 建议的实现顺序

1. 先建立 watchlist 与素材池
2. 再建立 Topic Radar 与 Top 5 建议
3. 再建立 Draft Pack 模板体系
4. 再接发布队列
5. 最后看哪些平台值得接草稿箱/自动化

## 17.3 为什么这样排

因为真正最难的不是“能不能发”，而是：

- 有没有值得发的题
- 有没有同行资本独特的切入角度
- 能不能稳定产出好内容
- 能不能形成复利型内容资产

## 18. 风险与约束

## 18.1 平台风险

- 反爬
- 登录态失效
- 验证码
- 账号风控
- 草稿箱不可自动写入

## 18.2 内容风险

- 观点空泛
- 跟风搬运
- 事实不准
- 引用来源不清
- 结论过强但证据不足

## 18.3 组织风险

- 创始人没有时间看太碎的信息
- 如果建议单太长、太乱，会降低使用意愿
- 如果稿件一稿多投严重，会损害品牌质感

### 18.4 因此必须坚持的原则

1. 人类始终保留选题确认权
2. 人类始终保留最终发布权
3. 系统必须保留原始来源链接
4. 系统必须输出“理由”，不能只报热点
5. 系统必须为不同平台做差异化适配

## 19. 待确认问题清单

以下问题建议后续逐步定下来：

1. 同行资本对外内容的主要受众是谁？
   - 创业者？投资人？产业方？媒体？潜在 LP？

2. 第一阶段最重要的三个平台是哪三个？

3. 同行资本希望内容偏哪种风格？
   - 深度行研
   - 热点评述
   - 公司/项目观察
   - 创始人观点
   - 混合型

4. 选题建议每天几次最合适？
   - 早报一次
   - 早晚两次
   - 热点触发式补报

5. 微信、小红书、知乎是否都要在第一阶段直接产出成稿？

6. 哪些平台后续真的值得接草稿箱，而不是仅做稿件包？

7. 是否需要建立“同行资本内容风格手册”？

8. 是否需要把历史优质文章、历史表达、品牌素材导入系统作为训练基础？

## 21. 一句话总结

这套系统的目标，不是做一个“会发稿的机器人”，而是搭建一个：

> **能持续发现值得写的话题、能给出有判断的选题建议、能批量生产多平台内容、并逐步形成同行资本内容资产与市场影响力的内容运营系统。**

如果这个方向确认无误，下一步就可以进入：

1. 首版 watchlist 设计
2. Top 5 建议单模板设计
3. Draft Pack 模板设计
4. agent 工作流与开发拆解
