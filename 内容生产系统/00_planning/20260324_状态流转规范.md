# 同行资本市场内容系统｜状态流转规范

## 1. 文档目的

这份文档用于冻结第一阶段各核心对象的状态流转规则，避免后续出现：

- 同一对象状态名字不一致
- 有人把“已批准”理解成“已发布”
- 有人把“draft_ready”理解成“已经进队列”

---

## 2. 主流程总览

第一阶段主流程统一按下面理解：

```text
watchlist_source
  → source_packet
  → topic_cluster
  → topic_candidate
  → approved_topic
  → draft_pack
  → publish_queue_item
  → performance_review
```

---

## 3. 分对象状态流转

## 3.1 `watchlist_source`

### 合法状态

- `candidate`
- `active`
- `paused`
- `archived`

### 流转规则

```text
candidate → active
candidate → archived
active ↔ paused → archived
```

### 说明

- `candidate`：待观察源，尚未正式纳入监控盘
- `active`：正常监控
- `paused`：暂时停用，后续可恢复
- `archived`：历史保留，不再恢复为正式源

---

## 3.2 `source_packet`

### 合法状态

- `captured`
- `normalized`
- `clustered`
- `archived`

### 流转规则

```text
captured → normalized → clustered → archived
```

### 说明

- `captured`：刚抓到，未经整理
- `normalized`：已转成可消费文本
- `clustered`：已归入某个话题簇
- `archived`：只保留历史记录

---

## 3.3 `topic_cluster`

### 合法状态

- `open`
- `scored`
- `promoted`
- `parked`
- `archived`

### 流转规则

```text
open → scored → promoted
open → scored → parked
parked → promoted
promoted → archived
parked → archived
```

### 说明

- `open`：刚形成话题簇
- `scored`：已完成基础打分
- `promoted`：已被提升为正式候选题来源
- `parked`：暂时观察
- `archived`：关闭

---

## 3.4 `topic_candidate`

### 合法状态

- `drafted`
- `pending_human`
- `approved`
- `parked`
- `expired`

### 流转规则

```text
drafted → pending_human → approved
drafted → pending_human → parked
pending_human → expired
parked → pending_human
approved → archived-by-topic
```

### 说明

- `drafted`：已写好候选卡片，但未正式送审
- `pending_human`：等待创始人确认
- `approved`：被选中，进入正式写稿链
- `parked`：暂不写，但未来可能再启用
- `expired`：热点窗口过去

> 注：`approved` 后不在 `topic_candidate` 里继续流转，而是切到 `approved_topic` 对象。

---

## 3.5 `approved_topic`

### 合法状态

- `approved`
- `drafting`
- `draft_ready`
- `queued`
- `published`
- `reviewed`

### 流转规则

```text
approved → drafting → draft_ready → queued → published → reviewed
drafting → approved
draft_ready → drafting
queued → draft_ready
```

### 说明

- `approved`：刚确认，尚未正式写稿
- `drafting`：Draft Pack 正在生产
- `draft_ready`：草稿包完成，待入发布队列
- `queued`：已生成一个或多个发布项
- `published`：至少已有一个平台发出
- `reviewed`：该话题已完成复盘

---

## 3.6 `draft_pack`

### 合法状态

- `drafting`
- `needs_revision`
- `ready`
- `superseded`

### 流转规则

```text
drafting → ready
drafting → needs_revision → drafting
ready → superseded
```

### 说明

- `drafting`：正在生产多平台稿件
- `needs_revision`：需要返修
- `ready`：可用于发布排队
- `superseded`：已有新版本

---

## 3.7 `publish_queue_item`

### 合法状态

- `queued`
- `waiting_human_publish`
- `published`
- `deferred`
- `cancelled`

### 流转规则

```text
queued → waiting_human_publish → published
waiting_human_publish → deferred
deferred → waiting_human_publish
queued → cancelled
waiting_human_publish → cancelled
```

### 说明

- `queued`：已排入队列
- `waiting_human_publish`：等待人类手动发
- `published`：已发
- `deferred`：延期
- `cancelled`：取消

---

## 3.8 `performance_review`

### 合法状态

- `scheduled`
- `collecting`
- `ready`
- `closed`

### 流转规则

```text
scheduled → collecting → ready → closed
ready → collecting
```

### 说明

- `scheduled`：已排复盘
- `collecting`：正在收集表现数据
- `ready`：可输出复盘结论
- `closed`：本轮复盘结束

---

## 4. 关键触发规则

## 4.1 从 `topic_candidate` 到 `approved_topic`

只有在出现明确人工确认时，才能创建 `approved_topic`：

- 选第 N 个
- 只做某几个平台
- 改成某种角度再写

没有明确确认，不允许自动越级写正式稿。

---

## 4.2 从 `draft_pack` 到 `publish_queue_item`

只有当 `draft_pack.status = ready` 时，才允许生成发布项。

否则：

- 返修中的 draft 不准排队
- 半成品不准直接进入发布队列

---

## 4.3 从 `publish_queue_item` 到 `performance_review`

只有实际发布后，才允许进入正式复盘。

若未发布：

- 可以做预估，不可以写正式 review

---

## 5. 第一阶段必须避免的错误流转

以下流转一律视为错误：

1. `topic_candidate.pending_human → draft_pack`
2. `approved_topic.approved → publish_queue_item`
3. `draft_pack.drafting → published`
4. `queued → reviewed`
5. `source_packet.captured → topic_candidate`

---

## 6. 一句话总结

市场内容系统第一阶段的状态机，从今天起就按这份文档执行；后续 skill、模板、bot 通知、控制台展示，都应该严格沿用这些状态定义。
