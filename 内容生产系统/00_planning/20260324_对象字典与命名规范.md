# 同行资本市场内容系统｜对象字典与命名规范

## 1. 文档目的

这份文档用于冻结市场内容系统第一阶段的核心对象定义，避免后续在写 skill、做流程、落文件时，每个人对“对象是什么、字段有哪些、怎么命名”理解都不一样。

第一阶段先冻结 8 个核心对象：

1. `watchlist_source`
2. `source_packet`
3. `topic_cluster`
4. `topic_candidate`
5. `approved_topic`
6. `draft_pack`
7. `publish_queue_item`
8. `performance_review`

---

## 2. 命名总原则

### 2.1 一个对象同时保留 3 层标识

每个对象都保留以下三层：

1. **object_id**：系统内部唯一 ID
2. **object_key**：人可读、可检索的稳定键
3. **title**：给人看的标题

### 2.2 命名格式原则

- `object_id`：短前缀 + 时间戳 + 随机尾巴或序号
- `object_key`：`日期__主题域__核心短语`
- `title`：自然语言标题

### 2.3 文件命名原则

建议统一为：

```text
YYYYMMDD_HHMMSS__object_key__object_type.md
```

例如：

```text
20260324_173000__ai_infra_inference_shift__topic-candidate.md
```

### 2.4 平台命名枚举

第一阶段统一使用以下平台值：

- `wechat`
- `xiaohongshu`
- `zhihu`
- `x`
- `bilibili`
- `toutiao`
- `baijiahao`
- `youtube`
- `tiktok`
- `douyin`
- `web`
- `rss`

## 3. 对象字典

## 3.1 `watchlist_source`

### 作用

定义“系统每天要看哪些源”。

### 一句话理解

它不是一篇内容，而是一个长期被监控的内容入口。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `source_id` | 唯一 ID |
| `source_key` | 稳定键，例如 `x__sama` |
| `source_name` | 人可读名称 |
| `platform` | 平台类型 |
| `source_type` | 媒体 / 竞品 / KOL / 公司 / 榜单 / 趋势入口 |
| `handle_or_url` | 账号名、频道名或 URL |
| `language` | 主要语言 |
| `region` | 区域，例如 `cn` / `global` |
| `priority` | `p0` / `p1` / `p2` |
| `topic_tags` | 关联主题标签 |
| `signal_quality` | 信号质量判断 |
| `citation_reliability` | 引用可靠性判断 |
| `capture_method` | 默认抓取方式 |
| `status` | `candidate` / `active` / `paused` / `archived` |
| `owner` | 当前由谁维护 |
| `notes` | 备注 |

### 状态说明

- `candidate`：待观察，尚未进入正式盘
- `active`：正式监控
- `paused`：暂时不抓
- `archived`：保留记录，不再监控

---

## 3.2 `source_packet`

### 作用

承接一次具体抓取结果。

### 一句话理解

它是“从某个源拿回来的某一条原始内容材料”。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `packet_id` | 唯一 ID |
| `packet_key` | 稳定键 |
| `source_id` | 对应 `watchlist_source` |
| `source_name` | 来源名 |
| `source_type` | 社区讨论 / 媒体稿 / 官方 listing / mirror / 趋势入口 |
| `platform` | 来源平台 |
| `region` | `cn` / `global` / `mixed` |
| `source_url` | 原始链接 |
| `title` | 原始标题 |
| `author_or_channel` | 作者 / 频道 / 账号 |
| `published_at` | 原始发布时间 |
| `captured_at` | 系统抓取时间 |
| `language` | 内容语言 |
| `content_type` | 文章 / 视频 / 推文串 / 帖子 / 榜单项 |
| `canonical_url` | 规范化后的最终链接 |
| `capture_method` | 实际抓取方式 |
| `normalization_method` | 整理方式 |
| `translation_needed` | 是否需要翻译进入中文工作流 |
| `signal_quality` | 该 source 对发现选题的信号质量 |
| `primary_source` | `yes` / `partial` / `no` |
| `verification_status` | `primary-source` / `official-platform-listing` / `secondary-report` / `mirror-signal` / `community-signal` / `fallback-entry` |
| `raw_text_path` | 原文或转写落盘路径 |
| `raw_capture_path` | 原始抓取快照路径 |
| `distilled_body` | 清洗后的可消费正文 |
| `summary` | 简短摘要 |
| `quote_candidates` | 可引用片段 |
| `topic_tags` | 初步打标 |
| `heat_hint` | 高热 / 早信号 / 背景材料判断 |
| `status` | `captured` / `normalized` / `clustered` / `archived` |

### 状态说明

- `captured`：刚抓到，尚未整理
- `normalized`：已整理成可消费文本
- `clustered`：已归入某个话题簇
- `archived`：仅保留，不再继续处理

---

## 3.3 `topic_cluster`

### 作用

把大量零散素材归并成一个“话题簇”。

### 一句话理解

它回答的是：这些素材其实都在讲同一件事。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `cluster_id` | 唯一 ID |
| `cluster_key` | 稳定键 |
| `title` | 话题簇标题 |
| `topic_domain` | 主题域，例如 AI / 半导体 / 量子 |
| `core_angle` | 当前归并后的主角度 |
| `packet_ids` | 关联的 `source_packet` 列表 |
| `platform_coverage` | 覆盖平台列表 |
| `heat_signals` | 热度/传播证据 |
| `evidence_signals` | 证据/原始来源证据 |
| `market_score` | 市场热度综合评分 |
| `relevance_score` | 与同行资本相关性评分 |
| `brand_fit_score` | 与同行资本主身份的贴合度评分 |
| `timeliness_score` | 时效性评分 |
| `writeability_score` | 可写性评分 |
| `battlefield_type` | `core` / `adjacent` / `out_of_scope` |
| `adjacency_reason` | 若为邻接题，说明它如何反向影响主战场 |
| `differentiation_angle` | 为什么这题该我们写、差异化切口是什么 |
| `risk_flags` | 风险标签 |
| `status` | `open` / `scored` / `promoted` / `parked` / `archived` |

### 状态说明

- `open`：刚归并
- `scored`：已打分
- `promoted`：已升级为正式候选题
- `parked`：先观察，不进入当天候选
- `archived`：归档

---

## 3.4 `topic_candidate`

### 作用

承接“准备给创始人看的正式候选题”。

### 一句话理解

它是从话题簇中挑出来、准备进入 Top 5 建议单的对象。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `candidate_id` | 唯一 ID |
| `candidate_key` | 稳定键 |
| `cluster_id` | 来源 `topic_cluster` |
| `title` | 候选题标题 |
| `one_line_judgment` | 一句话判断 |
| `why_now` | 为什么现在值得做 |
| `market_potential` | 市场侧看点，例如热度/增速/讨论度 |
| `heat_signal` | 热度与发酵信号总结 |
| `evidence_signal` | 证据强度总结 |
| `brand_fit_judgment` | 品牌侧判断，说明为何推荐或不优先推荐 |
| `competition_view` | 内容竞品 / 产品竞品 / 话题竞品态势 |
| `why_us` | 为什么该同行资本写、我们的差异化切口 |
| `suggested_angle` | 建议切入角度 |
| `suggested_platforms` | 建议平台 |
| `battlefield_type` | `core` / `adjacent_high_heat` / `adjacent_watch` |
| `adjacency_reason` | 若非主战场，说明它如何反哺主战场 |
| `evidence_links` | 核心原始链接 |
| `risks` | 风险提示 |
| `priority_rank` | 在当日建议单中的排名 |
| `status` | `drafted` / `pending_human` / `approved` / `parked` / `expired` |

### 状态说明

- `drafted`：已生成但尚未送审
- `pending_human`：已送到创始人待选择
- `approved`：被选中
- `parked`：先放一边，可能后续再用
- `expired`：过了时效窗口

---

## 3.5 `approved_topic`

### 作用

承接创始人已确认的选题。

### 一句话理解

它回答的是：这次正式要写哪一个题、按什么角度、做哪些平台版本。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `topic_id` | 唯一 ID |
| `topic_key` | 稳定键 |
| `candidate_id` | 来源候选题 |
| `title` | 正式选题标题 |
| `approved_angle` | 最终确认角度 |
| `requested_platforms` | 要输出的平台 |
| `special_instructions` | 人类追加要求 |
| `approved_by` | 由谁确认 |
| `approved_at` | 确认时间 |
| `status` | `approved` / `drafting` / `draft_ready` / `queued` / `published` / `reviewed` |

### 状态说明

- `approved`：刚确认
- `drafting`：正在写稿
- `draft_ready`：草稿包已完成
- `queued`：已进入发布队列
- `published`：至少已有一项发布
- `reviewed`：完成复盘

---

## 3.6 `draft_pack`

### 作用

承接一个选题的多平台草稿资产。

### 一句话理解

它不是一篇稿，而是一整套可发布内容包。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `draft_id` | 唯一 ID |
| `draft_key` | 稳定键 |
| `topic_id` | 来源 `approved_topic` |
| `wechat_path` | 微信稿路径 |
| `xiaohongshu_path` | 小红书稿路径 |
| `zhihu_path` | 知乎稿路径 |
| `x_path` | X 稿路径 |
| `title_options` | 标题候选 |
| `summary_options` | 摘要 / 导语候选 |
| `cover_or_visual_notes` | 配图建议 |
| `citation_block` | 引用来源清单 |
| `revision_notes` | 修订说明 |
| `status` | `drafting` / `needs_revision` / `ready` / `superseded` |

### 状态说明

- `drafting`：正在生成
- `needs_revision`：需要返修
- `ready`：可进入发布队列
- `superseded`：被新版替代

---

## 3.7 `publish_queue_item`

### 作用

承接每个平台维度的待发布动作。

### 一句话理解

同一个 `draft_pack` 可以拆成多个待发布项，一个平台一条。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `queue_id` | 唯一 ID |
| `queue_key` | 稳定键 |
| `topic_id` | 来源选题 |
| `draft_id` | 来源草稿包 |
| `platform` | 目标平台 |
| `content_path` | 对应稿件路径 |
| `publish_owner` | 由谁实际发布 |
| `planned_publish_at` | 计划发布时间 |
| `actual_publish_at` | 实际发布时间 |
| `publish_url` | 已发布链接 |
| `status` | `queued` / `waiting_human_publish` / `published` / `deferred` / `cancelled` |
| `notes` | 备注 |

### 状态说明

- `queued`：已排入待发布
- `waiting_human_publish`：等待人类手动发
- `published`：已发
- `deferred`：延期
- `cancelled`：取消

---

## 3.8 `performance_review`

### 作用

承接一次内容复盘结果。

### 一句话理解

它不是原始数据，而是对一次内容表现的判断与经验总结。

### 必备字段

| 字段 | 含义 |
| --- | --- |
| `review_id` | 唯一 ID |
| `review_key` | 稳定键 |
| `topic_id` | 对应选题 |
| `queue_ids` | 关联发布项 |
| `review_window` | 复盘窗口，例如 `24h` / `7d` |
| `platform_metrics` | 各平台核心数据 |
| `adoption_result` | 题材采纳效果 |
| `title_learnings` | 标题层结论 |
| `platform_fit_learnings` | 平台适配结论 |
| `source_learnings` | 来源质量结论 |
| `next_actions` | 下次怎么调整 |
| `status` | `scheduled` / `collecting` / `ready` / `closed` |

### 状态说明

- `scheduled`：已计划复盘
- `collecting`：在收集数据
- `ready`：已可阅读
- `closed`：结案归档

## 4. 对象关系

第一阶段对象关系统一按下面理解：

```text
watchlist_source
  └── source_packet (1:n)
        └── topic_cluster (n:1)
              └── topic_candidate (1:1 or n:1)
                    └── approved_topic (0:1)
                          └── draft_pack (1:1)
                                └── publish_queue_item (1:n)
                                      └── performance_review (n:1 or 1:1 per window)
```

## 5. 第一阶段平台优先级冻结

第一阶段正式平台冻结为以下 7 个：

1. `wechat`
2. `xiaohongshu`
3. `zhihu`
4. `x`
5. `bilibili`
6. `toutiao`
7. `baijiahao`

说明：

- 这 7 个平台都可合法出现在 `approved_topic.requested_platforms`
- 但不是每个题都必须全选
- 平台选择遵循“按题目 fit 选择平台束”

以下平台保留为扩展来源或后续目标平台，但不进入第一阶段图文发布主链路：

- `youtube`
- `tiktok`
- `douyin`

## 6. 一句话总结

从今天开始，市场内容系统第一阶段的核心对象，就按这份文档来定义；后续所有 skill、模板、目录、状态流转，都不应该再偏离这套对象字典。
