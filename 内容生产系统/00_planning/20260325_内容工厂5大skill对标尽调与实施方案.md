## 3. 从热门 skill 提炼出的结论

### 3.1 市场上强的 skill，几乎都不是“自动写一篇稿”

而是至少做对了下面其中几件事：

1. 先理解业务上下文
2. 明确用户是谁
3. 明确平台差异
4. 明确输出物长什么样
5. 明确复盘怎么做

---

### 3.2 我们现在最缺的，不是“更多采集 skill”

在内容工厂里，采集链已经有：

- `topic capture`
- `topic radar`
- `approved topic`
- `draft pack`
- `content polish`
- `publish queue`
- `performance review`

当前最大的缺口在中后段：

- 题有了，但包装不够强
- 稿有了，但冷启动不够友好
- 内容写了，但没有按用户层次翻译
- 稿件有了，但没有真正渲染成平台终稿
- 发完之后，没有把经验系统性吃回去

---

### 3.3 我们的 5 个 skill，必须比热门 skill 更贴业务

如果只做成“一个缩水版 marketingskills + 一个缩水版 baoyu-skills”，那没有意义。

因此，这 5 个 skill 的目标不是泛化，而是：

> **比热门 skill 更懂“同行资本的题材、用户、平台、业务目标和内容链路”。**

---

## 4. 我们准备落地的 5 个 Skill

### Skill 1｜`th-market-hook-title-cover`

#### 解决什么问题

- 为什么有人看到标题就不点
- 为什么点进来的人一开始不愿意继续读
- 为什么封面和标题没有形成合力

#### 适用节点

- `approved_topic -> draft_pack`
- `draft_pack -> polish`

#### 核心输入

- `approved_topic`
- `source refs`
- `target platform`
- `target audience`
- 相关原始事件背景

#### 核心输出

- 标题候选（按平台）
- 封面文案候选
- 首屏 opening 候选
- 钩子类型标注
- “为什么这组更容易点”的解释

#### 必须内置的能力

1. 钩子类型选择
2. 标题公式库
3. 封面信息压缩规则
4. 冷启动可理解性检查
5. 禁止标题和正文脱节

#### 对标热门 skill

| 对标对象 | 它的优点 | 它对我们不够的地方 | 我们必须赢在哪 |
| --- | --- | --- | --- |
| `marketingskills/social-content` | 有 hook formulas、platform best practices、viral reverse engineering | 更偏泛社媒运营，不聚焦我们这类“AI 热点 + 判断型内容” | 要把 `事件背景 + 主张 + 封面 + 首屏` 作为一体生成 |
| `marketingskills/copywriting` | 标题、CTA、页面结构方法扎实 | 更偏 landing page / conversion copy | 要专门适配“内容分发标题”，不是页面文案 |
| `baoyu-cover-image` | 封面图生成能力很强，维度丰富 | 更偏视觉生成，不管题是否好点、开头是否承接 | 要把“能点开”和“图是否好看”统一起来 |
| `meta-ads-creative` | 很重 hook、格式适配、创意 testing | 主要面向广告素材 | 我们要做的是公域内容，不是投放广告 |

#### 结论

这不是“起标题 skill”，而是：

> **包装层的第一道门。**

---

### Skill 2｜`th-market-context-bridge`

#### 解决什么问题

- 读者不知道原始事件是什么
- 稿子默认读者已经懂背景
- 一上来进入分析，冷启动用户掉得很快

#### 适用节点

- `draft_pack`
- `content polish`

#### 核心输入

- `approved_topic`
- 原始 source refs
- 目标平台
- 用户层次

#### 核心输出

- `context block`
- 一句话背景版
- 详细背景版
- 平台适配版背景说明

#### 必须内置的能力

1. 识别“这篇读者最可能不知道什么”
2. 用最短字数交代事件背景
3. 保持背景不抢主判断风头
4. 背景与主论点之间有自然桥接

#### 对标热门 skill

| 对标对象 | 它的优点 | 它对我们不够的地方 | 我们必须赢在哪 |
| --- | --- | --- | --- |
| `content-research-writer` | research / outline / citations / hook 改进整合得好 | 没有把“冷启动背景块”独立成高优先级结构 | 要专门解决“读者一头雾水”的问题 |
| `copywriter` | 很重 audience / voice / structure | 更偏营销页，不是新闻事件或行业动态承接 | 我们要擅长“复杂事件 -> 简明背景” |
| `content-strategy` | 很强的选题与内容结构逻辑 | 不直接负责每篇稿子的背景桥接 | 我们要把背景块产品化、模板化 |

#### 结论

这是让内容从“圈内人稿子”变成“公域可消费内容”的关键 skill。

---

### Skill 3｜`th-market-audience-translator`

#### 解决什么问题

- 同一篇内容，对小白太深，对专业人群又太浅
- 平台转换只是语气变化，没有做人群翻译
- 读者知道你很懂，但不知道与你何干

#### 适用节点

- `draft_pack -> repurpose`
- `platform rewrite`

#### 核心输入

- 原始 master draft
- 目标平台
- 目标人群标签

#### 核心输出

- 小白版
- 半专业版
- 泛投资 / 商业判断版
- 对应的“读者收益点”表达

#### 必须内置的能力

1. 识别原文默认的知识前置
2. 把术语翻译成用户听得懂的话
3. 不牺牲主判断，只改表达方式
4. 保留“这和你有什么关系”的利益点

#### 对标热门 skill

| 对标对象 | 它的优点 | 它对我们不够的地方 | 我们必须赢在哪 |
| --- | --- | --- | --- |
| `xiaohongshu-converter` | 平台语气、结构、emoji、标题规则很清晰 | 只解决“小红书化”，不解决多用户层次转换 | 我们要做“受众翻译 + 平台翻译”的双转换 |
| `ux-writing` | 非常重清晰、用户目标、文案可理解性 | 偏 UI 文案，不是长内容 / 热点内容 | 我们要把“清晰”扩展到长文和判断型内容 |
| `copywriter` | 强 brand voice 和 audience discovery | 更适合页面 copy | 我们要做的是内容版 audience fit |

#### 结论

这不是“改口语一点”，而是：

> **把同一判断，翻译成不同人群愿意继续读的版本。**

---

### Skill 4｜`th-market-platform-renderer`

#### 解决什么问题

- 现在很多稿子只是 Markdown 中间稿
- 没有真正渲染成 7 平台标准支持范围内的最终成品（公众号 / 小红书 / 知乎 / X / B站专栏 / 今日头条 / 百家号）
- 排版 knowhow 没有沉淀

#### 适用节点

- `content polish -> publish queue`

#### 核心输入

- polished master draft
- 平台要求
- 封面 / 配图 / context block / CTA

#### 核心输出

- 微信终稿
- 小红书终稿
- 知乎终稿
- X thread 终稿
- 配套 handoff 文件

#### 必须内置的能力

1. 把 Markdown 渲染成平台终稿结构
2. 平台特有的小标题 / 引用 / 分段规则
3. 封面 / 配图 / 图注 / 结尾 CTA 统一
4. 与发布器对接的 handoff 产物

#### 对标热门 skill

| 对标对象 | 它的优点 | 它对我们不够的地方 | 我们必须赢在哪 |
| --- | --- | --- | --- |
| `wechat-publisher` | 能一键转公众号格式并推草稿箱 | 解决的是“发”，不是“怎么排才更能读” | 我们要把排版策略前置到发布前 |
| `baoyu-post-to-wechat` | 对微信发布很实用 | 更偏单平台 | 我们要统一多平台渲染层 |
| `baoyu-article-illustrator` | 很强的图文增强能力 | 不负责整篇文章的平台排版闭环 | 我们要决定何处该加图、何处不该加 |
| `baoyu-infographic` / `baoyu-slide-deck` | 很强的成品化能力 | 偏视觉延展，不是核心分发正文 | 我们要先解决正文消费体验 |
| `md2x` / `markdown-slides` | 负责转换格式 | 对平台 knowhow 不够深 | 我们要做的是“平台终稿”，不是纯格式转换 |

#### 结论

这一步是从“写完了”升级到“能发了”的关键。

---

### Skill 5｜`th-market-postmortem-optimizer`

#### 解决什么问题

- 发完以后，只知道“好 / 不好”，不知道为什么
- 上一篇踩过的坑，下一篇还在踩
- 好内容没有形成模板复利

#### 适用节点

- `performance review`
- 下轮内容迭代前

#### 核心输入

- 平台表现数据
- 最终标题 / 封面 / opening
- 正文结构
- 互动评论
- 来源与发布时间

#### 核心输出

- 复盘结论
- 标题 / 封面 / hook 的得失
- 背景块是否过长 / 过短
- 用户评论的真实反馈
- 下一轮可复用规则
- 需要更新的模板 / 规则 / skill 片段

#### 必须内置的能力

1. 先诊断“点击问题”还是“留存问题”
2. 能把评论与互动纳入复盘，而不是只看表面数字
3. 形成结构化 learnings
4. 反推到下一轮标题 / 开头 / 排版 / 选题

#### 对标热门 skill

| 对标对象 | 它的优点 | 它对我们不够的地方 | 我们必须赢在哪 |
| --- | --- | --- | --- |
| `social-content` | 有 metrics、weekly review、reverse engineering | 更偏社媒账号运营常规动作 | 我们要把 learnings 写回内容工厂链路 |
| `analytics-tracking` | 强数据与埋点思维 | 更偏网站 / 增长分析 | 我们要对“内容成品”做 postmortem |
| `copy-editing` | 强七轮编辑法 | 是内容发布前，不是发布后复盘 | 我们要形成“发布后继续优化”的闭环 |

#### 结论

如果没有这个 skill，前面 4 个 skill 的提升无法复利。

## 5. 为什么不是先做 Brand Voice Humanizer

这个方向当然重要，但本轮不放进最优先的 5 个 skill 里，原因很简单：

> 当前内容工厂的更大瓶颈，不是“AI 味还太重”，而是“内容还没被包装成能被平台吃到结果的形态”。

所以：

- `brand voice / humanizer` 仍然是 P1
- 但 P0 先补“点击率、阅读时长、平台适配、复盘学习”

---

## 6. 这 5 个 Skill 必须满足的“不过线不施工”标准

### 6.1 对标线

如果做不到下面这些，就不该上线：

1. **不弱于 `marketingskills` 的结构化程度**
2. **不弱于 `baoyu-skills` 的输出物意识**
3. **不弱于 `xiaohongshu-converter` 的平台细则清晰度**
4. **不弱于 `wechat-publisher` 的 handoff / 发布衔接意识**

### 6.2 我们自己的必胜点

如果只是“和别人差不多”，也没意义；我们必须强在：

1. 更懂 AI / agent / 一人公司题材
2. 更懂同行资本的目标用户
3. 更懂我们自己的内容工厂流程
4. 更懂“选题 -> 草稿 -> 平台终稿 -> 复盘”的整链路

## 8. 建议施工顺序

### Phase 1｜先补“能拿结果”的三件套

1. `th-market-hook-title-cover`
2. `th-market-context-bridge`
3. `th-market-platform-renderer`

原因：

- 这 3 个直接决定点击率、阅读时长和最终发稿质量

---

### Phase 2｜再补“人群翻译 + 复盘复利”

4. `th-market-audience-translator`
5. `th-market-postmortem-optimizer`

原因：

- 这两项决定内容工厂能不能越跑越准，而不是一直靠人工盯

---

## 9. 这轮施工的产出要求

后续正式写 skill 时，每个 skill 至少要有：

1. `SKILL.md`
2. 触发条件
3. 输入输出规范
4. 失败边界
5. 至少一个 references 文件
6. 至少一个真实内容样例
7. 在内容工厂链路中的调用位置

---

## 10. 结论

这一轮 5 个 skill，不是为了把内容工厂做成“会写稿的大模型壳子”，而是为了补齐 5 个真正影响结果的业务能力：

1. 让内容更容易被点开
2. 让冷启动用户更容易看懂
3. 让不同用户都觉得“这和我有关”
4. 让稿子真正变成平台终稿
5. 让系统能从表现中反推经验

只要这 5 个补上，内容工厂才算真正从“会写”进入“会运营”。
