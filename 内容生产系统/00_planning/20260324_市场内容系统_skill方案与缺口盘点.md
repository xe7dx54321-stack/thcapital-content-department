## 2. 我对文章核心观点的理解

这篇文章真正有价值的地方，不是列了 7 个 skill，而是点出了一个非常重要的认知：

> 内容系统最值钱的，并不是“自动写稿”，而是让 agent 具备：**会找、会收、会改、会呈现、会补工具链** 的能力。

这和我们自己的业务诉求高度一致。

我们要做的也不是一个“自动发稿机器人”，而是一套：

1. 能持续发现题材和线索
2. 能从多平台收一手材料
3. 能把内容整理成适合我们品牌表达的形式
4. 能针对不同平台做结构化改写
5. 能逐步沉淀经验并形成复利的系统

所以，这篇文章对我们最大的启发，不是“装哪几个 skill”，而是：

> **市场内容系统必须由多个 skill 组成一条工作链，而不是只有一个写稿 skill。**

## 4. 对每个外部 skill 的判断

## 4.1 Agent Reach

### 这是什么

`Agent Reach` 的核心价值，不是某个单功能，而是把多个内容平台的访问能力统一打通：

- X / Twitter
- Reddit
- YouTube
- GitHub
- Bilibili
- 小红书
- 抖音
- LinkedIn
- Boss 直聘
- RSS
- 任意网页

它本质上是一个**跨平台内容入口总线**。

### 我们现在有没有

**有，而且已经装了。**

本地已有：
- `/Users/apple/.openclaw/skills/agent-reach/SKILL.md`

它的能力已经覆盖了内容雷达系统最关键的一层：

- 平台接入
- 平台状态检查
- Cookie / proxy / browser-based 配置
- 对多个平台直接调用上游工具

### 对我们值不值

**非常值，而且是底层基础设施。**

如果没有这层，后面很多“找热点”“找竞品内容”“跨平台追踪发酵度”的需求就会变成空中楼阁。

### 结论

- 状态：**已具备**
- 重要性：**P0 必需**
- 动作：**不需要重复造轮子，直接复用**

---

## 4.2 youtube-ultimate

### 这是什么

`youtube-ultimate` 的核心价值是：

- 抓 YouTube 字幕
- 下载视频
- 搜索与探索 YouTube 内容
- 不依赖官方 quota

它适合把长访谈、发布会、频道更新变成可进一步处理的原材料。

### 我们现在有没有

**部分有。**

我们当前已经具备：

1. `agent-reach` 中的 YouTube 能力
2. `yt-dlp` 路线
3. `baoyu-youtube-transcript` 这类更适合转字幕/转文本的能力可直接借鉴

也就是说，我们并不是完全没有 YouTube 能力，而是：

> **已经有基本能力，但还没有把它收成市场内容系统的一条标准工作流。**

### 对我们值不值

**值，但不是最稀缺的缺口。**

因为它更多是“内容源补强”，不是“内容判断与转化”的核心 skill。

### 结论

- 状态：**能力部分已有，尚未产品化收束**
- 重要性：**P1**
- 动作：
  - 不需要把 `youtube-ultimate` 当唯一方案
  - 可以直接复用现有 `agent-reach + yt-dlp + transcript` 能力
  - 后续在市场系统里把它收敛成“视频情报采集子流程”即可

## 4.3 Humanizer

### 这是什么

`Humanizer` 的核心价值，是让已有文本：

- 降低 AI 味
- 改得更自然
- 保留判断，不乱加水话

### 我们现在有没有

**没有对应的正式 skill。**

我们现在当然可以让模型“改自然一点”，但那不等于正式 skill。

当前缺的是：

- 明确的风格边界
- 明确的去 AI 味规则
- 明确的禁止项
- 明确的“保留观点 / 不加空话 / 不腻歪 / 不鸡汤”纪律
- 和同行资本品牌语气绑定的表达打磨方式

### 对我们值不值

**非常值，而且非常重要。**

但这里要注意：

> 我们不应该直接追求一个泛化的 Humanizer，而应该做一个 **同行资本品牌风格版 Humanizer**。

因为“去 AI 味”本身不是目的，目的是：

- 保留信息密度
- 保留判断力
- 保留品牌气质
- 让成稿更像“同行资本在说话”

### 结论

- 状态：**没有正式 skill，是明显缺口**
- 重要性：**P0 必需**
- 动作：
  - 不建议直接照搬泛 Humanizer
  - 建议做成 `th-content-polish` 或 `th-brand-voice-polish`

## 4.4 ppt-generator

### 这是什么

这个 skill 的定位很清楚：

- 把讲稿转成乔布斯风、极简、竖屏 HTML 演示稿
- 输出单个可运行 HTML 文件
- 强节奏、强留白、极简科技风

### 我们现在有没有

**没有直接同款。**

但我们已有市场系统需求里，这一类并不是第一优先。

### 对我们值不值

**值，但不是 P0。**

它更适合以下场景：

- 创始人分享
- 对外演讲
- 内容二次传播
- 把长文改成演示内容

这对品牌影响力有帮助，但不是市场内容系统的最早瓶颈。

### 结论

- 状态：**未具备同款能力**
- 重要性：**P2 可选增强**
- 动作：
  - 后续如果你要强化“文章 → 演讲 / 分享 / 路演”的复用链路，再接入
  - 当前不建议作为第一批重点施工

## 4.5 frontend-slides

### 这是什么

它本质上也是演示内容生成，只是更偏：

- 更完整的 HTML slides
- 更丰富的动画与表现力
- 更像一整套动态演示能力

### 我们现在有没有

**没有直接同款。**

### 对我们值不值

**有价值，但与 `ppt-generator` 高度重叠。**

在我们的市场内容系统早期，这两者都不是核心矛盾。

### 结论

- 状态：**未具备**
- 重要性：**P2 / P3**
- 动作：
  - 暂不优先
  - 将来在“内容资产复用”阶段二选一即可，不需要两套都上

## 4.6 baoyu-skills

### 这是什么

这不是一个 skill，而是一整个内容工具箱。关键点不在于“全装”，而在于**按需拆用**。

我重点看了其中几个和我们最相关的子 skill：

1. `baoyu-url-to-markdown`
2. `baoyu-youtube-transcript`
3. `baoyu-infographic`
4. `baoyu-slide-deck`
5. `baoyu-post-to-wechat`
6. `baoyu-xhs-images`

### 我们现在有没有

#### 已有或近似已有

- `baoyu-url-to-markdown`：我们已有 `jina-reader`、`x-reader-wechat`、网页抓取链路，**能力近似已有**
- `baoyu-youtube-transcript`：我们已有 YouTube transcript 路线，**能力近似已有**

#### 没有但很有价值的

- `baoyu-xhs-images`
- `baoyu-post-to-wechat`
- `baoyu-infographic`

#### 可后置的

- `baoyu-slide-deck`

### 我的判断

`baoyu-skills` 最大价值不是“帮我们写稿”，而是补齐内容成品化的最后几步：

- 网页内容沉淀为 markdown
- YouTube 内容转字幕
- 内容转小红书图文系列
- 内容发布到公众号
- 内容转信息图

这正好对应我们市场内容系统中最容易缺的一层：

> **从“选题/写稿”到“成品化/分发”的能力层。**

### 结论

- 状态：**部分能力已有，部分是高价值缺口**
- 重要性：
  - `url-to-markdown`：P1（近似已有）
  - `youtube-transcript`：P1（近似已有）
  - `xhs-images`：P1（很值得）
  - `post-to-wechat`：P1（很值得）
  - `infographic`：P2（有价值）
  - `slide-deck`：P2（可后置）
- 动作：**不建议整包搬运，建议按子 skill 选择性吸收**

## 4.7 Marketing Skills for AI Agents

### 这是什么

这套技能集合不是“内容生产工具箱”，而更像一套**营销工作方法 skill 库**。

它的价值在于流程与框架，而不是具体平台抓取。

我重点看了几个最相关的子 skill：

1. `product-marketing-context`
2. `content-strategy`
3. `social-content`
4. `copy-editing`

### 我们现在有没有

#### 没有，但非常重要

- `product-marketing-context`
- `content-strategy`
- `social-content`
- `copy-editing`

我们现在市场系统里最大的隐患之一，就是：

> 如果没有明确的“品牌上下文 / 内容策略 / 平台适配 / 编辑打磨” skill，系统最后很容易变成一个只会追热点和写稿的流水线。

### 为什么它重要

这几个 skill 提供的不是“工具”，而是**内容运营的方法论骨架**：

- 先定义品牌和受众上下文
- 再定义内容支柱与内容策略
- 再定义社交平台的表达方式
- 最后对已有文案做多轮编辑打磨

这和我们自己的诉求高度契合。

### 但哪些我们现在不需要

这个 repo 里还有很多 skill：

- paid ads
- CRO
- revops
- pricing
- lead magnets
- popup CRO
- referral program
- email sequence
- launch strategy

这些都不是同行资本市场内容系统当前阶段的主战场。

### 结论

- 状态：**方法论层明显缺失**
- 重要性：
  - `product-marketing-context`：**P0**
  - `content-strategy`：**P0**
  - `social-content`：**P0**
  - `copy-editing`：**P0/P1**
- 动作：
  - 不建议整包引入
  - 建议把这几个方法论 skill “本土化改造”为同行资本版

## 5. 我们当前已经有了什么

如果站在“市场内容系统”视角看，我们现在其实已经有了一个很强的**底层能力层**，只是缺少上层的内容策略与表达 skill。

### A. 多平台入口能力

已有：
- `agent-reach`
- `x-reader-wechat`
- `jina-reader`
- `multi-search-engine`
- `reddit-scraper`
- `podcast-capture`

这意味着：

- 我们并不是从 0 开始
- 内容系统最难的“跨平台抓入口”这件事，已经有基础

### B. 业务 skill 化能力

已有：
- `topdown-industry-research`
- `topdown-signal-supply`
- `topdown-project-evaluation`

## 6. 我们真正缺的是什么

我认为目前最缺的，不是抓取工具，而是下面这几类 **上层 market-content skill**。

## 6.1 缺口一：品牌与受众上下文 skill

### 问题

现在如果让 agent 去做内容，它很容易犯两个问题：

1. 不知道同行资本到底想对谁说话
2. 不知道同行资本的表达边界和品牌气质是什么

### 对应 skill 方向

- 借鉴：`product-marketing-context`
- 我们要做：`th-market-brand-context`

### 这个 skill 要解决什么

- 同行资本的品牌定位
- 目标受众
- 平台角色分工
- 语气与风格
- 禁写项 / 风险项
- 哪些内容值得发，哪些不值得发

### 重要性

**P0**

没有这一层，所有写稿 skill 都会漂。

---

## 6.2 缺口二：选题雷达与话题筛选 skill

### 问题

有多平台入口，不等于有选题能力。

真正难的是：

- 海量内容里哪些值得看
- 哪些是热点噪音
- 哪些与同行资本相关
- 哪些值得今天写
- 哪些适合微信，哪些更适合小红书

### 对应 skill 方向

- 借鉴：`Agent Reach` + `content-strategy` + `social-content`
- 我们要做：`th-topic-radar`

### 这个 skill 要解决什么

- Watchlist 巡检
- 热点归并
- 多平台发酵度判断
- 选题评分
- Top 5 候选建议单生成
- 给出理由和原始链接

### 重要性

**P0**

这是市场内容系统的第一核心 skill。

---

## 6.3 缺口三：平台差异化 Draft Pack skill

### 问题

“写一篇稿”不等于“完成内容生产”。

我们真正需要的是：

- 微信版
- 小红书版
- 知乎版
- X 版
- 后续可能还有视频脚本版

### 对应 skill 方向

- 借鉴：`social-content`
- 我们要做：`th-draft-pack`

### 这个 skill 要解决什么

- 同一个主题，针对不同平台重写
- 保留核心判断，不做机械一稿多投
- 输出标准化 Draft Pack
- 保留原始引用来源
- 标出标题候选、风险点、建议配图

### 重要性

**P0**

这是从“选题建议”走向“真正可发布内容”的关键 skill。

---

## 6.4 缺口四：品牌风格打磨 / 去 AI 味 skill

### 问题

即便写出了一稿，如果没有品牌化编辑，很容易出现：

- AI 味重
- 套话多
- 节奏平
- 像资讯总结，不像品牌表达

### 对应 skill 方向

- 借鉴：`Humanizer` + `copy-editing`
- 我们要做：`th-content-polish`

### 这个 skill 要解决什么

- 去 AI 味
- 保留信息密度
- 保留同行资本判断感
- 统一风格
- 禁止空话、鸡汤、废话、油腻表达
- 可做多轮 sweep：清晰度 / 观点力度 / 证据 / 节奏 / 风格

### 重要性

**P0**

这是市场内容系统的第二核心 skill。

---

## 6.5 缺口五：内容成品化与分发适配 skill

### 问题

内容系统如果只停在文字，会很快遇到瓶颈：

- 小红书图文怎么出
- 公众号草稿怎么落
- 文章怎么改信息图
- 长文怎么改成分享稿

### 对应 skill 方向

- 借鉴：`baoyu-xhs-images` / `baoyu-post-to-wechat` / `baoyu-infographic`
- 我们要做：`th-content-repurpose` + `th-publish-ops`

### 这个 skill 要解决什么

- 长文转小红书图卡
- 长文转信息图
- 稿件写入发布队列
- 后续条件成熟时再接公众号草稿箱

### 重要性

**P1**

这很重要，但应该排在选题和成稿之后。

---

## 6.6 缺口六：发布复盘与经验回灌 skill

### 问题

如果每次写完就结束，系统不会变聪明。

我们需要知道：

- 哪类题采纳率高
- 哪个平台更适合哪种表达
- 哪种标题更容易通过
- 哪些监控源更有效

### 对应 skill 方向

- 借鉴：`content-strategy` / `social-content` 的复盘思路
- 我们要做：`th-content-review`

### 这个 skill 要解决什么

- 发布后效果记录
- 周 / 月复盘
- 高表现题材归因
- 低表现题材归因
- 反向修正 topic radar 与 draft pack

### 重要性

**P1**

不是第一个上线，但必须纳入体系设计。

6. `th-publish-ops`
7. `th-content-review`

### 结论

> 我们真正缺的是**市场内容专用的上层工作方法 skill**，而不是底层抓取能力。

---

## 7.3 可借鉴但不必原样引入

1. `youtube-ultimate`
2. `Humanizer`
3. `baoyu-skills` 的部分子 skill
4. `Marketing Skills for AI Agents` 的部分方法论 skill

### 结论

> 最优策略不是“整包安装”，而是**抽取思想 + 选择性复用 + 本土化改造成同行资本版 skill**。

---

## 7.4 当前阶段不需要或不应优先的

1. paid ads
2. 各类 CRO
3. referral program
4. pricing strategy
5. revops
6. 冷邮件 / 销售 enablement
7. 全自动多平台发布
8. 花大精力做复杂视频制作链路
9. 过早做 PPT / slides 双套系统

### 结论

> 这些要么离当前目标太远，要么会把系统拉进复杂而低回报的方向里，不适合现在优先投入。

---

## 8. 同行资本市场内容系统的推荐 skill 架构

我建议最终 skill 架构不是一个 skill，而是 7 个 skill 组成的一条链。

## 8.1 Skill-01：`th-market-brand-context`

### 作用

定义同行资本在市场内容系统中的基础上下文。

### 主要内容

- 品牌定位
- 目标受众
- 平台定位
- 内容边界
- 风格与语气
- 禁止项
- 可引用 / 不可引用的信息类型

### 对标外部 skill

- `product-marketing-context`

### 优先级

**P0**

---

## 8.2 Skill-02：`th-topic-radar`

### 作用

把多平台输入转化为 Top 5 选题建议。

### 主要内容

- watchlist 巡检
- 热点聚类
- 发酵度判断
- 选题评分
- 给出建议标题、理由、平台建议、原始链接

### 对标外部 skill

- `Agent Reach`
- `content-strategy`
- `social-content`

### 优先级

**P0**

---

## 8.3 Skill-03：`th-source-capture-and-citation`

### 作用

把热点相关原始材料转成结构化可引用素材。

### 主要内容

- URL 转 markdown
- YouTube transcript
- 微信文章正文抓取
- 播客 / 视频 / 长文素材抽取
- 统一引用编号与链接保留

### 对标外部 skill

- `baoyu-url-to-markdown`
- `baoyu-youtube-transcript`
- `x-reader-wechat`
- `jina-reader`

### 优先级

**P0/P1**

> 严格说，底层能力已有，但需要在市场系统里正式工作流化。

---

## 8.4 Skill-04：`th-draft-pack`

### 作用

把一个已确认选题转成多平台草稿包。

### 主要内容

- 微信版
- 小红书版
- 知乎版
- X 线程版
- 标题备选
- 导语与摘要
- 配图建议
- 引用来源清单

### 对标外部 skill

- `social-content`

### 优先级

**P0**

---

## 8.5 Skill-05：`th-content-polish`

### 作用

让成稿更像同行资本，而不是更像模型。

### 主要内容

- 去 AI 味
- 品牌风格统一
- 信息密度不掉
- 观点更干净有力
- 多轮编辑 sweep

### 对标外部 skill

- `Humanizer`
- `copy-editing`

### 优先级

**P0**

---

## 8.6 Skill-06：`th-content-repurpose`

### 作用

把长文变成适合平台传播的成品。

### 主要内容

- 微信长文 → 小红书图卡
- 微信长文 → 信息图
- 微信长文 → 演示稿 / 分享稿
- 微信长文 → 视频脚本

### 对标外部 skill

- `baoyu-xhs-images`
- `baoyu-infographic`
- `ppt-generator`
- `frontend-slides`

### 优先级

**P1/P2**

---

## 8.7 Skill-07：`th-publish-ops-and-review`

### 作用

把内容从“写完”推进到“可发、可管、可复盘”。

### 主要内容

- 发布队列
- 发布前检查单
- 草稿箱接入（后置）
- 发布提醒
- 表现记录
- 经验回灌

### 对标外部 skill

- `baoyu-post-to-wechat`
- `social-content` 的复盘思路

### 优先级

**P1**

---

## 9. 我建议的实际落地顺序

## 第一阶段：先把最小闭环跑通

先做 4 个 skill：

1. `th-market-brand-context`
2. `th-topic-radar`
3. `th-draft-pack`
4. `th-content-polish`

### 为什么

因为这 4 个 skill 一旦落地，就已经能跑出最小闭环：

- 找题
- 给建议
- 人工确认
- 出稿
- 打磨

这已经足够开始真实跑业务。

---

## 第二阶段：补齐素材与引用工作流

再做：

5. `th-source-capture-and-citation`

### 为什么

底层抓取我们有，但要形成稳定引用链，还要专门 skill 化。

---

## 第三阶段：补成品化与分发

再做：

6. `th-content-repurpose`
7. `th-publish-ops-and-review`

### 为什么

这是锦上添花，但不应该反过来抢走前面核心环节的优先级。

---

## 10. 最终判断：我们该怎么站在巨人的肩膀上

我的结论很明确：

### 10.1 不要做的事

1. 不要把文章里推荐的 skill 一股脑全装上
2. 不要重复造底层抓取轮子
3. 不要直接照搬泛营销 / 泛内容 skill
4. 不要一开始就把重点放到自动发稿和花哨呈现上

### 10.2 应该做的事

1. 复用现有底层抓取能力
2. 借鉴 `product-marketing-context / content-strategy / social-content / copy-editing` 的方法论骨架
3. 借鉴 `Humanizer` 的目标，但做成同行资本版品牌打磨 skill
4. 借鉴 `baoyu-skills` 的内容成品化能力，但按子 skill 选择性接入
5. 最终沉淀成一套**同行资本市场内容专用 skill 体系**

---

## 11. 一句话总结

这轮盘点之后，我的结论是：

> **我们不缺“上网找内容”的工具，缺的是把同行资本品牌表达、选题判断、多平台改写、内容打磨、成品化分发这些工作方法，正式 skill 化。**

所以接下来最合理的动作，不是继续装更多外部 skill，而是：

> **开始写同行资本自己的市场内容 skill 套件。**

建议下一步顺序：

1. 先写 `th-market-brand-context`
2. 再写 `th-topic-radar`
3. 再写 `th-draft-pack`
4. 再写 `th-content-polish`

等这 4 个定下来，市场内容系统就真正有骨架了。
