# 同行资本市场内容系统｜2026-03-25｜OpenAI News 与 X 稳定抓取方案

## 1. 为什么要单独定方案

这两条线都很重要，但之前都卡在“直接访问不顺手”：

- `OpenAI News`：direct 容易遇到 Cloudflare
- `X`：direct 虽然能开页面，但常常只有壳，没有足够有效内容

这轮补测后，已经不需要再把它们当成“理论上重要、实际上跑不动”的 B 类来处理。

---

## 2. `OpenAI News` 当前稳定方案

### 默认抓取路径

- 入口页：`Jina reader`
- 正文页：从列表拿到单篇 OpenAI 原文链接后，再继续抓对应文章

### 为什么这个方案成立

- 已验证可拿到新闻列表
- 可稳定识别标题、日期、文章链接
- 不必再依赖 direct html 抗 Cloudflare

### 运行规则

1. 先抓 `news` 列表
2. 识别当天/近期新增文章
3. 只对候选文章做单篇深入抓取
4. 正式引用时，链接仍然回到 OpenAI 原文，而不是 reader 中间层

---

## 3. `X` 当前稳定方案

### 默认抓取路径

- 默认：`Jina reader`
- 增强：保留 `x-auth` 作为 fallback

### 为什么这个方案成立

- 已验证 `@OpenAI`
- 已验证 `@karpathy`
- 同类 `X profile` 路径对官方号 / builder 号可复用

### 运行规则

1. 把 `X` 当成**信号发现源**
2. 先拿 profile 与近期帖文摘录
3. 如果是观点类内容：
   - 可以直接进入选题线索层
4. 如果是事实类内容：
   - 必须继续顺着帖子里的官方链接、博客、repo、docs 回链

### 不要误用

- 不要把 `X` 的一句帖子直接当成最终硬证据
- 不要因为帖子热就跳过后续 official chain

## 5. 这两条线的正确角色

### `OpenAI News`

- 角色：官方大事件、正式口径、产品与治理信息入口
- 适合：
  - 重大版本发布
  - 产品能力变化
  - 官方战略动作

### `X`

- 角色：更快的社交信号层
- 适合：
  - 观点
  - 快讯
  - builder 侧讨论
  - 官方社交补充口径

---

## 6. 最终判断

这两条线当前已经具备明确、可执行的 daily 方案：

> `OpenAI News` 负责“官方正式更新”，`X` 负责“社交快信号”，两者配合，而不是互相替代。
