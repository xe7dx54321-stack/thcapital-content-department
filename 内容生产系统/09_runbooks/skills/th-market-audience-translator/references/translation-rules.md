# Translation Rules

Use these rules to translate one TH Capital judgment across audience layers.

## 1. Preserve the invariant

Always keep:

- the object
- the event
- the thesis
- the source-grounded risk

Only change:

- how quickly things are defined
- what benefit frame comes first
- what analogy or example is used
- what consequence is emphasized

## 2. Translate by priority, not by simplification alone

Different audiences care about different first questions.

So translation should first change:

- the order of explanation

before changing:

- wording

## 3. Terminology rule

For every important term:

1. decide if it must remain
2. if yes, translate it once in plain language
3. then use the term consistently

Example:

- `agent harness`
  - Layer A: `一套用来组织多个 agent 协作的底座`
  - Layer B: `多 agent 的编排底座`
  - Layer C: `agent orchestration / operating layer`

## 4. Payoff rule

The payoff line should match the layer:

- Layer A: `这会影响你以后怎么找机会 / 用工具 / 做项目`
- Layer B: `这会影响你怎么搭 workflow / 选 stack / 判断是否值得做`
- Layer C: `这会影响价值会积在哪一层，以及谁更有议价权`

## 5. Avoid false audience adaptation

Bad adaptation:

- beginner version = empty encouragement
- operator version = stacked nouns, no mechanism
- strategic version = all framework, no event anchor

Good adaptation:

- same thesis
- different doorway
- different consequence frame
