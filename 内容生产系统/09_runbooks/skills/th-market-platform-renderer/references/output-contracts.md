# Output Contracts

This file defines what “renderer output” should look like in TH Capital content operations.

## 1. WeChat HTML handoff

Should include:

- source markdown path
- final title
- optional subhead / lead
- first-screen plan
- emphasis candidates
- quote block candidates
- image insertion suggestions
- source / risk retention note
- manual production note

## 2. Cover / visual brief

Should include:

- core title
- core judgment
- cover claim
- visual metaphor suggestion
- what must not mislead
- optional side text / subcopy

## 3. Xiaohongshu card brief

Should include:

- cover title
- card 1 conclusion
- card 2-4 breakdown
- final card CTA
- tone / visual guidance

## 4. Zhihu final rendering note

Should include:

- answer-first summary
- context bridge location
- section list
- evidence / source note placement

## 5. X thread layout note

Should include:

- line 1 claim
- line 2 context
- line 3-5 breakdown
- final line follow-up / CTA

## 6. Bilibili final rendering note

Should include:

- practical promise in the opening
- section / step breakdown
- where examples or pitfalls are inserted
- what makes the piece feel community-native

## 7. Toutiao final rendering note

Should include:

- final title
- first 2 short paragraphs
- short-block structure
- where大众表达 replaces abstract wording

## 8. Baijiahao final rendering note

Should include:

- search-intent title
- answer-first opening
- keyword-aware subheads
- source / risk retention note

## 9. Publish readiness note

Should state:

- whether the render handoff is complete
- what still requires human work
- whether the remaining work is packaging-only or content-quality-related
