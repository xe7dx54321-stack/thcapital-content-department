- `target_platform`: `wechat`
- `target_audience`: `AI 兴趣小白 + 半专业从业者`
- `core_claim`: `真正能赚钱的 computer-use agent，不是会点网页，而是能接 legacy 系统并在真实环境里稳定自愈。`
- `cold_start_gap`: `大多数读者不知道 Minicor 是谁，也不知道“computer-use”与“生产级桌面自动化”之间的差别。`

## Opening Hook Options

### Option A

很多人看到 computer-use agent，第一反应还是：这不就是“AI 帮你点网页”吗？  
但这次 Minicor 真正值得看的，不是演示效果，而是它开始去接那些没有 API、但企业又最愿意付钱解决的 legacy 系统。

### Option B

如果一个 agent 只能在浏览器里演示得很聪明，它未必值钱。  
真正值钱的，是它能不能进企业最脏、最老、最难改的系统里，把流程稳定接起来。Minicor 这次，开始有点那个味道了。

### Option C

这条线看起来像一条普通的 YC launch，但它其实把一个很重要的判断打实了：  
下一阶段真正能赚钱的 agent，拼的不是“会不会自动操作”，而是“能不能进生产环境”。

## Recommended Package

- `title`: `真正能赚钱的 agent，不是会点网页的那种`
- `cover`: `会点网页，不等于能赚钱`
- `opening`: `Option B`

## Why This Package Wins

- 标题先给结论，再制造认知落差
- 封面把“演示能力”和“商业价值”区分开
- 开头没有立刻堆背景，而是先讲用户能理解的价值判断
- 背景可以在下一段补：`Minicor 是 YC 新近出现的一家公司，主打 self-healing desktop automation for legacy enterprise systems`

## What To Avoid

- 不要写成 `Minicor 发布了一个很厉害的 product`
- 不要标题只写 `Minicor：self-healing desktop automations`
- 不要开头 3 段都在解释什么是 computer use
