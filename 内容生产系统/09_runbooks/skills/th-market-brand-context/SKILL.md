# th-market-brand-context

Use this skill for any **同行资本公域内容** task, including:

- deciding whether a topic fits the brand
- choosing the right public angle
- adapting one topic across platforms
- rewriting drafts into TH Capital voice
- reviewing whether copy is too empty, too academic, or too salesy
- adding soft CTA hooks that help attract the right target users

Important boundary:

- This skill is **not** a source-whitelisting tool.
- Do not use “main battlefield vs adjacent battlefield” to suppress source intake.
- Source capture should stay broad inside the AI universe.
- This skill applies at the **topic selection, recommendation, drafting, and review** stages.

The content universe is **AI only**.

The **main battlefield** is:

- AI agents
- autonomous operations
- one-person-company cases
- skills / workflows / stacks
- TH Capital's own build in public

Adjacent AI topics are allowed only when they clearly change:

- agent capability
- agent cost / efficiency
- agent infra reliability
- agent interaction form
- agent business feasibility or distribution

This skill governs **public-domain expression only**. Private-domain operation, community maintenance, and follow-up conversion are human-owned. The only private-domain concern inside this skill is whether public content leaves a natural next step for the right people.

## Core workflow

1. Confirm the task type:
   - topic planning
   - title / angle design
   - first draft
   - rewrite / polish
   - review / QA

2. Confirm the content target:
   - which platform
   - which audience layer
   - what outcome matters most now:
     - spread
     - clarity
     - trust
     - soft conversion

2A. If the task is topic selection or ranking, split the work into two passes:
   - Pass 1: market-first scoring (heat, growth, discussion, writeability, clarity, hook strength)
   - Pass 2: brand-fit re-ranking (main battlefield fit, adjacency value, identity reinforcement, drift risk)

3. Decide whether the topic belongs in public scope.
   - If it is not in AI, reject it.
   - If it is core battlefield content, mark it as naturally high fit.
   - If it is adjacent AI content, keep it in the pool when it is genuinely hot or strategically important, then judge whether it materially affects agents, autonomous operations, one-person-company execution, or workflow design.
   - If the value depends on private interviews, sensitive judgments, or unfinished deep research, do not over-expose it in public.
   - Convert it into a public-safe version: screening, explanation, trend signal, or misconception correction.

4. Choose one main content model:
   - 热点快拆
   - 机会清单
   - 误区纠偏
   - 赛道入门解释

5. Draft or rewrite with these rules:
   - explain first, guide second
   - keep high information density
   - keep language clear and restrained
   - avoid fake certainty
   - sound like an informed operator, not a hype account

6. Add CTA only when it helps.
   - default to one primary CTA
   - keep it soft
   - target the right user type
   - never promise materials or access you do not actually have

7. Run the final check:
   - Can users understand the point in 10 seconds?
   - Is there a clear “why this matters”?
   - Is this more than a news restatement?
   - Does it clearly strengthen TH Capital's main battlefield identity?
   - If this is an adjacent topic, did we explicitly explain its impact on agents / workflows / autonomous operations?
   - Does it fit the platform role?
   - Is the tone clearly TH Capital rather than generic AI copy?
   - If there is a CTA, does it feel natural and proportionate?

8. If producing a ranked topic list, do not hide strong adjacent topics.
   - Keep them visible.
   - Mark why they are hot.
   - Then explain why some battlefield-aligned topics are recommended ahead of them.

## Hard constraints

- Do not write as if TH Capital already has fully formed closed-door insight when it does not.
- Do not drift into generic AI news coverage with no agent or workflow relevance.
- Do not turn battlefield logic into an upstream source-class hierarchy.
- Do not use exaggerated growth-hack language.
- Do not sound like a generic marketing account or a training-course seller.
- Do not turn probability judgments into certainty.
- Do not expose private-domain-only value in public just to make the piece feel more “advanced”.

## Output guidance

When the task is **planning**, output:

1. content goal
2. target audience
3. best content model
4. platform fit
5. market potential
6. brand-fit judgment
7. recommended angle
8. optional CTA direction
9. risks / what to avoid

When the task is **drafting or rewriting**, output:

1. final draft
2. optional title options
3. optional CTA line
4. brief note on why this version fits the platform

When the task is **reviewing**, explicitly flag:

- too empty
- too academic
- too much AI tone
- too much sales pressure
- too weak on “why now”
- too weak on target-user attraction
