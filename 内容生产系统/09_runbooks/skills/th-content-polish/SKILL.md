# th-content-polish

Use this skill for Step 7 work:

- draft pack polishing
- brand voice alignment
- de-AI rewrite
- title / hook improvement
- publish readiness review

## Important boundary

- This skill works **after** a Draft Pack already exists.
- It does **not** replace topic approval.
- It does **not** replace platform rewriting from Step 6.
- It does **not** permit dropping citations for the sake of smoother copy.

Its job is:

> 让成稿更像同行资本在说话，而不是像模型在背稿。

## Sweep sequence

Apply the sweeps in order. Do not skip straight to cosmetic edits.

### Sweep 1: clarity

- remove fuzzy transitions
- make the main point visible earlier
- shorten bloated sentences
- keep one paragraph, one job
- make the object / stakes / why now land early enough

### Sweep 1.5: long-form completeness

- for WeChat / Zhihu / Bilibili / Baijiahao / Toutiao, verify the draft can answer:
  - 这到底是什么事
  - 为什么现在值得看
  - 这跟谁有关
  - 我们的判断是什么
- if the draft jumps into analysis before the reader knows the event, fix that first
- add an early proof anchor before the article becomes abstract

### Sweep 2: de-AI

- remove template-like filler
- remove needless empathy prefaces
- remove “让我们看看 / 值得注意的是 / 在当下这个节点” style padding
- remove rhetorical inflation

### Sweep 3: stronger judgment

- make TH Capital’s real view explicit
- avoid sitting on the fence unless uncertainty is real
- keep judgment tied to evidence

### Sweep 3.5: personality consistency

- sound like TH Capital, not generic AI media
- keep information density
- keep risk language specific and human
- do not turn “de-AI” into empty colloquial chatter
- if a `style route` is present, borrow only the declared layers
- run the corresponding creator pack `style-audit.md` before marking the pack ready
- if the copy feels more like the external creator than TH Capital, pull it back immediately

### Sweep 4: hook and title

- make the opening worth continuing
- ensure the title says what the reader gets
- avoid empty hype
- title / cover / opening optimization is the business role's own job, not a supervisor afterthought
- fix the current draft's packaging before considering topic replacement
- prefer `object + action/change + reader payoff/risk`
- reject half-sentences, ellipsis titles, pure abstract judgment, and titles that hide the object too long

### Sweep 4.5: visual-package consistency

- title / cover / first-screen promise must point to the same core claim
- do not let the cover oversell a different story from the body
- ensure the cover brief gives a human publisher a real starting point

### Sweep 5: citation retention

- keep source refs
- keep the risk note
- make sure strong claims still map back to sources
- keep at least one early proof anchor in long-form platforms

### Sweep 6: platform fit

- WeChat: clearer structure, stronger mid-depth argument
- Xiaohongshu: shorter, sharper, lower threshold
- Zhihu: answer-first, then explain
- X: high signal density, no long essay spillover
- Bilibili: more process detail, more builder tone, more concrete caveats
- Toutiao: more direct hook, more大众表达, but no cheap clickbait
- Baijiahao: clearer keyword anchoring, stronger search readability, no keyword stuffing

## Prohibited patterns

- empty “big opportunity is coming” language
- chicken soup / motivational fluff
- “AI tone” padding
- pseudo-formality
- hype unsupported by the source refs
- smoothing away the actual risk note

## Output guidance

When polishing a Draft Pack, update:

1. platform drafts
2. title options if needed
3. summary options if needed
4. polish checklist
5. publish readiness note
6. long-form completeness notes if a long-form platform is involved
7. cover / visual brief if the existing brief is too thin

If the pack is still weak after polishing:

- mark `needs_revision`
- explain the blockers plainly
