# th-style-router

Use this skill when the task needs:

- 选题确定后的风格路由
- 不同题材对应哪个公众号打法
- 决定 primary / secondary overlay
- 防止写稿时临时乱拼风格

## Load order

1. Read `../../../08_brand_assets/learning_knowledge_assets/latest__style-router.md`
2. Read `../../../08_brand_assets/learning_knowledge_assets/latest__learning-knowledge-index.md`
3. Read the relevant creator pack `skill-card.md` in `../../../08_brand_assets/learning_knowledge_assets/creator_skill_packs/`
4. If you are about to write正文, read the same creator pack's `text-patterns.md`
5. If title / cover / inline visuals matter, read the same creator pack's `visual-patterns.md`
6. Before final routing advice, read the same creator pack's `style-audit.md`
7. Read the selected `approved_topic` or live draft

## Core workflow

1. Judge the content route first:
   - 方法论 / workflow / skill / agent 教程
   - 工具体验 / builder 拆解
   - 大热点解释 / 冷启动事件稿
   - 产业判断 / 商业分析
   - 技术前沿 / 研究传播

2. Select:
   - `primary overlay`
   - `secondary overlay`
   - optional `support overlay`
   - the exact `overlay skill` to load at draft time

3. Lock the borrowed layers explicitly:
   - title / cover
   - opening hook
   - context bridge
   - evidence placement
   - paragraph rhythm
   - visual pacing

4. State what must remain TH Capital core:
   - judgment boundary
   - citation discipline
   - risk note
   - platform fit

## Required output

1. `content_route`
2. `primary_overlay`
3. `secondary_overlay`
4. `support_overlay`
5. `overlay_skill_to_load`
6. `borrowed_layers`
7. `brand_core_to_keep`
8. `what_not_to_copy`
9. `why_this_route_fits`
