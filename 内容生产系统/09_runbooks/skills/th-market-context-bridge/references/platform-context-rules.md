# Platform Context Rules

These rules tell you how much context to give and where to place it.

## 1. WeChat

### Best use

- hook first
- context soon after
- argument follows

### Working rule

- let the first `1-2` paragraphs create interest
- then bridge the missing event/object context
- then enter the main argument

### Common mistake

- opening claim is strong
- then 4 paragraphs of abstraction
- reader still does not know what company / product / event we are talking about

---

## 2. Xiaohongshu

### Best use

- context must be tiny
- translate object into human language

### Working rule

- usually one sentence is enough
- if the topic is obscure, use one extra line

### Common mistake

- trying to teach the whole background
- turning the first screen into a glossary

---

## 3. Zhihu

### Best use

- answer first
- context second
- then reasoning

### Working rule

- the context bridge should make the object concrete fast
- but it should not dominate the answer

### Common mistake

- answering the question too late
- background paragraph sounds like copied media report

---

## 4. X

### Best use

- the context is compressed into the claim or the next line

### Working rule

- if the object is niche, name it clearly
- never make users read 3 lines before they know the subject

### Common mistake

- hot take first, unclear object later

---

## 5. Cross-platform heuristic

If a reader says:

- “我不知道这是什么”

then the context is missing.

If a reader says:

- “你前面铺垫太久了”

then the context is too heavy or placed too early.
