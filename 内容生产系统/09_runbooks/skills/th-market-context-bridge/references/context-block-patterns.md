# Context Block Patterns

Use these patterns to make TH Capital content understandable to cold-start readers.

The goal is not to retell the whole event.  
The goal is to bridge the minimum gap between the hook and the real argument.

## 1. Object-first pattern

### Use when

- the reader likely does not know the company / product / protocol / tool
- the event object is obscure

### Pattern

`[对象] 是什么 + 最近发生了什么 + 为什么它值得现在被讨论`

### Example skeleton

`[对象] 是一家 / 一个 [一句话定义]。最近它做了 [事件]，这件事之所以值得看，不是因为新闻本身，而是因为它暴露了 [后续关键变量]。`

---

## 2. Change-first pattern

### Use when

- the object is relatively known
- the real problem是“到底变了什么”

### Pattern

`过去是什么 + 这次变化是什么 + 变化为什么重要`

### Example skeleton

`过去大家更多把它看成 [旧定位]。但这次它往 [新定位] 迈了一步，这意味着接下来更值得看的已经不是 [旧问题]，而是 [新问题]。`

---

## 3. Stakes-first pattern

### Use when

- the event is abstract
- the reader needs to quickly feel relevance

### Pattern

`这事表面看起来是 X + 但真正影响的是 Y + 下面解释为什么`

### Example skeleton

`这件事表面像 [表层新闻]，但真正被改变的其实是 [用户 / 商家 / 创作者 / agent 系统] 的 [某个环节]。先把背景讲清楚，再看为什么这比表面重要得多。`

---

## 4. Misread-correction pattern

### Use when

- public consensus is likely wrong
- the opening already用了反常识 hook

### Pattern

`大家容易把它看成 X + 实际上它更接近 Y + 这里先交代一下它到底是什么`

### Example skeleton

`很多人会把这件事理解成 [误读]。但更准确地说，它是 [准确解释]。先把这层背景放平，后面再看为什么它会指向更大的变化。`

---

## 5. What a good context block always does

A good context block should answer:

1. the object
2. the event
3. why now
4. how it connects to the coming argument

## 6. What a weak context block looks like

- starts with vague scene-setting
- says “最近 AI 圈有件事” but never makes the object concrete
- dumps too much history
- explains the wrong layer
- never bridges to the argument
