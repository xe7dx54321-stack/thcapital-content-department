# 同行资本内容部门项目状态

> 本文件用于维护当前开发状态，避免每轮小步施工后丢失上下文。

## 当前目标

把现有“脚本驱动的内容工厂”逐步升级为稳定的 AI / Agent 领域内容生产 Agent 系统。

目标主线：

1. 每天自动化获取 AI 和 Agent 领域的一手高价值信息。
2. 将时效信息结构化进入系统。
3. 基于规则和模型判断信息价值。
4. 将高价值信息转化为微信公众号文章、小红书推文等成品内容。
5. 通过头部自媒体学习和人工反馈持续迭代选题与表达能力。

## 当前系统判断

当前项目已经具备：

- 内容工厂控制台入口。
- 活跃市场内容系统目录。
- 多类信源抓取脚本。
- 官方更新 lane。
- 微信 / RSS 刷新脚本。
- 深度抓取脚本。
- 头部媒体学习 memo 和学习池看板。
- 前台 Markdown / HTML / JSON 快照产物。

当前主要问题：

1. 信息采集端不稳定，缺少信源健康监控、fallback、状态库和失败复盘。
2. 信息价值判断不稳定，缺少标准化 evidence packet、topic cluster 和多维评分体系。
3. 内容生成质量不稳定，缺少 brief → outline → draft → editor → fact-check 的分阶段流水线。
4. 头部内容学习还停留在看板层，需要沉淀为可执行的 pattern library。
5. 现有代码仍存在较多本机路径和脚本式运行方式，需要逐步工程化。

## 开发策略

后续开发采用小步施工：

- 每次只完成一个明确 checkpoint。
- 每次都更新本文件和任务清单。
- 优先新增旁路能力，少直接改动核心生产脚本。
- 每一步都要有可执行验收命令。
