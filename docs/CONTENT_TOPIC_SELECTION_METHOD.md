# Content Topic Selection Method

## 目标

选题方法论回答一个问题：什么题值得同行资本写。

系统不只看“信息是否重要”，还要看它是否能形成可被读者记住、可被证据支撑、能产生独立判断的文章。

## 八个核心维度

- 变化强度：是否代表真实变化，而不是普通更新。
- 预期差：事实是否改变读者或市场原有预期。
- 产业链影响：是否影响模型、算力、工具、应用、渠道或客户预算。
- 证据强度：是否有一手来源和可核验证据。
- 叙事张力：是否有冲突、误读、转折或反方。
- 当前窗口期：为什么现在写。
- 独立判断潜力：能否形成一句可争论的核心判断。
- 读者收益：读完是否提升判断质量。

## 一票否决

以下选题即使热度高，也应谨慎或拒绝：

- 只有标题党、没有证据。
- 只是营销稿，没有实质变化。
- 中文媒体已经反复写烂，没有新判断。
- 缺少一手来源且无法补证据。
- 只有融资/PR，没有产品、技术或商业实质。
- 无法形成一句核心判断。
- 与 AI/Agent/产业/投资主线弱相关。

## 写作准入问题

进入写作链路前，选题必须能回答：

- 这个题在讲什么变化？
- 为什么现在写？
- 有什么预期差？
- 影响哪条产业链或哪类公司？
- 核心判断一句话是什么？
- 至少 3 条证据是什么？
- 反方或不确定性是什么？
- 读者读完获得什么？

## 使用方式

`config/topic_selection_methodology.json` 是机器可读配置。`make methodology-topic-score` 会用这套方法论旁路评估 high-value candidates，不替代原 value scoring，也不自动改规则。
