# 公众号文章图片策略方法

## 核心原则

图片不是装饰，而是信息表达工具。每张图必须服务文章核心判断、证据链、结构理解或读者记忆点。

## 图片类型

- `cover_visual`：封面图 / 首图，承载核心判断。
- `concept_diagram`：概念解释图，解释抽象概念。
- `value_chain_map`：产业链 / 价值链图，解释影响对象。
- `timeline_chart`：时间线图，解释为什么现在写。
- `comparison_table_visual`：对比图，展示预期差或方案差异。
- `framework_diagram`：分析框架图，帮助读者复述结论。
- `process_flow`：流程图，解释任务、价值或信息流。
- `evidence_snapshot`：证据截图 / 来源截图说明，增强可信度。

## 评价标准

- 是否服务核心判断。
- 是否有信息量。
- 是否一眼看懂。
- 是否贴合叙事位置。
- 视觉层级是否清楚。
- 是否有来源或证据说明。
- 是否避免模板化无意义配图。
- 手机端是否清楚。
- 版权风险是否可控。
- 是否符合专业、克制、高级的内容气质。

## 获取路径

- `evidence_snapshot`：来源截图 / 官方图，需人工确认版权和出处。
- `internal_chart`：系统根据结构化数据生成简单图表或表格图。
- `generated_concept_visual`：生成概念图 prompt，不默认调用图片模型。
- `manual_design_request`：生成设计需求，由人工或设计工具处理。

## 禁止事项

- 不自动下载版权不明图片。
- 不默认调用图片生成模型。
- 不把图片文件提交进仓库。
- 图片生成和图片使用都需要人工确认。
