# 谷歌制裁OpenClaw：开发者怒怼Anthropic会先打招呼

> 整理日期：2026-03-10
> 标签：#OpenClaw #谷歌 #Anthropic #vibeCoding #2026-02

---

## 背景

最近频频登上新闻头条的OpenClaw，终于被「制裁」了一回。

本周一，谷歌宣布限制部分开发者使用旗下vibe Coding平台Antigravity，并指控他们「恶意使用」，此举在社交平台上引发了争议。

---

## 事件经过

### 谷歌的解释

谷歌表示：

- 发现恶意使用Antigravity后端的情况**大幅增加**
- 严重影响了用户的服务质量
- 需要快速切断未按预期使用产品的用户的访问权限

### 受影响用户

一些用户发现，在使用以下服务后失去Google帐户访问权限：

1. 开源自主AI工具**OpenClaw**
2. 基于Antigravity构建智能体
3. 将OpenClaw智能体连接到自己的Gmail帐户

### 技术原因

- 大量后端Gemini token的访问
- 导致系统过载
- 降低普通用户的服务质量
- 谷歌不得不实行封禁

### 封禁范围

此次限制仅影响：
- Antigravity
- Gemini CLI
- Cloud Code Private API

**其他谷歌服务均保持运行**，且没有完整的谷歌帐户被封禁。

---

## 当事人回应

### Varun Mohan

Windsurf前联合创始人、现任Google Antigravity成员：

> "我们需要找到一种方法，快速切断这些未按预期使用产品的用户的访问权限。我们理解部分用户并不了解这种行为违反了我们的服务条款，我们会为他们提供恢复访问权限的途径，但我们的资源有限，我们希望公平对待所有正常用户。"

### Peter Steinberger

OpenClaw开发者：

> "谷歌的做法「相当严格」"

> "Anthropic在遇到类似问题时会直接联系我，而谷歌却「直接封禁」"

## 用户反馈

### 批评焦点

在谷歌官方论坛、Hacker News和Reddit等平台上，受影响的用户纷纷表达不满：

1. **缺乏预警**：没有提前通知
2. **沟通不畅**：处理流程不透明
3. **难以获得技术支持**：申诉困难

## 行业意义

### OpenClaw的崛起

OpenClaw是一个意外走红的开源实验

- 从默默无闻到成为vibe Coding代名词
- "人手一虾"成为极客标配

### 平台与开发者关系

1. **封禁的合理性**：平台有权保护服务质量
2. **沟通的重要性**：预警和申诉机制至关重要
3. **生态平衡**：如何在开放与管控之间取得平衡

### 开源VS闭源

- Anthropic：遇到问题会直接联系开发者
- 谷歌：直接封禁，缺乏沟通

两种不同的生态策略
