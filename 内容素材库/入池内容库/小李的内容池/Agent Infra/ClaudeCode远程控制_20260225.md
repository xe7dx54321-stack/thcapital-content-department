# Claude Code远程控制功能上线

> 整理日期：2026-03-11
> 标签：#Claude Code #Anthropic #AI工具 #远程控制 #2026-02

---

Claude Code刚刚推出了一项全新功能：远程控制。OpenClaw用户早就用上了这项功能，说实话，"发起任务，然后用手机查看"这种工作流程一旦尝试过就真的会上瘾。

现在，你可以在电脑终端跑起一个代码任务，然后拿起手机去散步或开会，随时随地接管任务进度。在此期间，Claude会在你的本地机器上持续运行，而你可以直接通过Claude App或claude.ai/code网页端控制整个会话。

该功能目前面向Max用户提供研究预览版，并即将向Pro用户开放。只需在终端输入 `claude rc` 即可快速上手。

## 核心亮点：设备在变，本地环境不变

远程控制功能打通了网页端、iOS和Android移动端与本地机器的连接。当你在本地启动远程会话时，所有进程都在本地运行，没有任何数据会被迁移到云端。

这意味着你可以实现以下操作：

- **调用完整的本地环境**：你的文件系统、MCP服务器、工具和项目配置均保持可用状态。

- **多端同步办公**：对话记录在所有连接设备上实时同步，你可以无缝穿插使用终端、浏览器和手机发送消息。

- **无惧中断**：如果你的笔记本电脑进入休眠或网络断开，只要机器重新上线，会话就会自动重连。

这与此前推出的网页版Claude Code有着本质区别。网页版运行在Anthropic托管的云基础设施上，而远程控制会话直接在你的本地机器上执行，与本地文件系统交互，网页和手机端只是监控和操作本地会话的窗口。

## 使用门槛

- **订阅要求**：必须是Pro或Max计划用户。目前不支持团队版、企业版，也不支持通过API密钥使用。

- **账号认证**：运行 `claude` 并使用 `/login` 命令完成网页端登录。

- **工作区信任**：至少需要在项目目录中运行过一次 `claude`，并接受工作区信任弹窗。

## 启动方式

**方式一：启动全新会话**

```python
claude remote-control
```

**方式二：从现有会话无缝切换**

```
/remote-control
```

或者使用缩写 `/rc`。

## 安全机制

本地Claude Code会话仅发起出站HTTPS请求，绝不会在你的机器上开放入站端口。流量通过Anthropic API基于TLS加密传输，并采用多种生命周期极短、按独立用途分配的凭证。
