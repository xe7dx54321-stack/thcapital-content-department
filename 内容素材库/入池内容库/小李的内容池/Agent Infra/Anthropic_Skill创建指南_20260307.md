## 核心内容

### 1. 什么是Skill？

"Skills，本质上是一套指令，被打包在一个简单的文件夹中，用于教会Claude如何处理特定的任务或工作流。通过Skills，用户无需在每次对话中重复解释自己的偏好、流程和专业知识，只需一次性教授，便可永久受益。"

"这正是定制化Claude最强大的方式之一。"

### 2. Skill的文件夹结构

一个skill是一个包含以下内容的文件夹：

- **SKILL.md (必需)**：包含YAML frontmatter和Markdown格式的指令
- **scripts/ (可选)**：可执行代码（如Python, Bash等）
- **references/ (可选)**：按需加载的文档
- **assets/ (可选)**：输出内容中使用的模板、字体、图标等资源

### 3. 核心设计原则

**渐进式披露 (Progressive Disclosure)**

"skill采用一个三级系统来管理信息加载："

- **第一级 (YAML frontmatter)**：这部分信息始终加载在Claude的系统提示中，让Claude知道何时应该使用某个技能
- **第二级 (SKILL.md主体)**：当Claude认为技能与当前任务相关时才会加载，包含完整指令和指导
- **第三级 (链接文件)**：技能目录中捆绑的额外文件，Claude可以根据需要自行导航和发现

"这种渐进式披露的设计，在保持专业能力的同时，最大限度地减少了Token的消耗。"

**可组合性 (Composability)**

"Claude可以同时加载多个技能。因此，你设计的技能应当能与其他技能良好共存，而不是假定它是唯一可用的能力。"

**可移植性 (Portability)**

"技能在Claude.ai、Claude Code和API中的工作方式完全相同。一次创建，即可在所有支持其依赖环境的平台上无修改地运行。"

### 4. Skill + MCP 厨房比喻

"MCP提供了专业厨房：它让你能够接触到工具、食材和设备。Skill则提供了菜谱：它包含创造有价值东西的详细步骤说明。"

| MCP (连接性) | Skill (知识) |
|-------------|-------------|
| 将Claude连接到你的服务 | 教会Claude如何高效使用你的服务 |
| 提供实时数据访问和工具调用能力 | 捕获工作流和最佳实践 |
| Claude能做什么 | Claude应该如何做 |

### 5. 没有Skill vs 拥有Skill

**没有Skill时**：
- 用户连接了MCP，但不知道接下来该做什么
- 每次对话都从零开始
- 由于用户每次提问方式不同，导致结果不一致

**拥有Skill后**：
- 预构建的工作流在需要时能自动激活
- 工具使用稳定、可靠
- 最佳实践融入每一次交互

### 6. Skill用例类别

**类别1：文档与资产创建**
用于创建一致、高质量的输出，包括文档、PPT、应用、设计、代码等。

**类别2：研究与分析**
用于系统地收集和分析信息。

**类别3：MCP增强型工作流**
与外部服务深度集成。

### 7. 规划与设计要点

**从用例开始**

"在编写任何代码之前，首先确定你的skill应该支持的2-3个具体用例。"

**好的用例定义示例**：
```
> 用例：项目冲刺规划
> 触发条件：用户说"帮我规划这次冲刺"或"创建冲刺任务"
> 步骤：
> 1. 从Linear获取当前项目状态 (通过MCP)
> 2. 分析团队速率和容量
> 3. 建议任务优先级
> 4. 在Linear中创建带有合适标签和预估时间的任务
> 结果：一个完全规划好的冲刺
```

## 技术要点

### 三级信息加载
1. **YAML frontmatter**：始终加载，知道何时调用
2. **SKILL.md主体**：相关时才加载
3. **链接文件**：按需加载

### 可组合性
- 技能可以同时加载多个
- 设计时考虑与其他技能共存

### 可移植性
- 一次创建，多平台运行
- Claude.ai、Claude Code、API完全一致
