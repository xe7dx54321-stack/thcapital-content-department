# Claude Code技术栈揭秘：2430次测试揭示工具偏好

> 整理日期：2026-03-10
> 标签：#ClaudeCode #AI编程 #技术栈 #2026-02

---

## 背景

向Claude Code提开发需求，却刻意不在prompt中提及任何具体工具，它会更倾向于选择用什么工具？又会展现出哪些偏好特征？

专注于量化AI主观决策的基准测试工作室Amplifying.ai，针对Claude Code的工具选择倾向开展了系统性研究。

---

## 研究方法

- **模型数量**：3款
- **项目类型**：4种
- **工具类别**：20个
- **测试次数**：累计分析**2430次**工具选择行为

### 实验设计

- 开放式提示词：如"我应该用什么？"
- 不包含任何工具名称
- 记录实际操作中的工具选择结果

---

## 核心发现

### 1. 倾向"自建"而非第三方工具

Claude Code更倾向于**自己编写自定义解决方案**，而不是直接推荐现成的第三方工具。

**数据**：
- 自定义/DIY实现占所有主要选择的**12%**
- 2073次中的**252次**是最常见的选择

### 2. 默认技术栈形成

Claude Code选择第三方工具时，会集中选：

| 类别 | 工具 |
|------|------|
| 部署 | Vercel |
| 数据库 | PostgreSQL |
| 支付 | Stripe |
| CSS | Tailwind CSS |
| UI组件 | shadcn/ui |
| 包管理 | pnpm |
| CI/CD | GitHub Actions |
| 监控 | Sentry |
| 邮件 | Resend |
| 状态管理 | Zustand |

### 3. 技术栈专属选择

根据不同技术栈选择专属工具：

| 技术栈 | 工具选择 |
|--------|----------|
| JS项目 | Drizzle（ORM） |
| Python项目 | SQLModel（ORM） |
| Next.js项目 | NextAuth.js（认证） |
| JS项目 | Vitest（测试） |
| Python项目 | pytest（测试） |
