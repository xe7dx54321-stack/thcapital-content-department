## 五大核心结论

### 1. 倾向"自建"而非第三方工具

Claude Code更倾向于**自己编写自定义解决方案**，而不是直接推荐现成的第三方工具。

- 自定义/DIY实现：**252次**（占12%）
- 成为最常见的选择

### 2. 默认技术栈已然形成

Claude Code选择第三方工具时，会集中选：

| 类别 | 首选工具 |
|------|----------|
| 部署 | Vercel |
| 数据库 | PostgreSQL |
| 支付 | Stripe |
| UI | Tailwind CSS、shadcn/ui |
| 包管理 | pnpm |
| CI/CD | GitHub Actions |
| 监控 | Sentry |
| 邮件 | Resend |
| 状态管理 | Zustand |

### 3. 部分工具类别已"锁定"单一工具

| 类别 | 主导工具 | 占比 |
|------|----------|------|
| CI/CD | GitHub Actions | **94%** |
| UI组件 | shadcn/ui | **90%** |
| 支付 | Stripe | **91%** |

### 4. 同一技术生态下，不同模型选择高度一致

- 三个模型在20个类别中的**18个**选择了相同首选工具
- 一致率达**90%**
- 仅有缓存和实时通信两个类别有真正分歧

### 5. 项目上下文比指令措辞更重要

- 同一工具类别在不同代码仓库中，Claude Code的选择会**随项目类型变化**
- 同一项目，用5种不同方式表述指令，选择稳定性达**76%**
