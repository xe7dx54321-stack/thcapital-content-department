*话题 01*

重要技术问题

1、 模型能力跃迁正在从 Verifiable 任务向 Fuzzy 任务迁移：

大模型能力的跃迁可以基于一个二维象限来理解，其中，纵轴是 “任务可被验证属性”，越向上，任务越难验证，横轴是 “解空间的大小（Solution Space）”，越向右，任务越开放、越长程，在这个象限划分下，模型能力跃迁可以分为三个阶段：

行业正将模型的能力从可验证任务（Crisp Task，如数学、代码）推向更模糊的任务（Fuzzy Task，如深度研究报告、商业计划书），沿着这个趋势，模型面对 fuzzy task 的自我进化能力很重要：当前模型在做 deep research、写 business plan 时表现不错，但它不关心结果是否真正成功——不像做数学题那样有明确的优化方向。未来 AI 要真正创造更多价值，需要能对开放任务做 self-criticizing、自我进化。

2、 Continual learning 是 26 年一个确定性的趋势：今天 OpenClaw 通过工程手段实现了一定程度的长程稳定，但真正的continual learning（lifelong learning）怎么实现仍旧需要在 2026 年持续探索。

• 核心问题：模型训练完之后，能否在部署阶段通过某种方式更新自己的内部状态（memory 或模型参数），让自己在特定用户或特定任务上越做越好？

• 现实难点：如果任务非常长程，则奖励信号非常稀疏。但这里可能存在范式创新的机会：

e.g 内在奖励机制（Intrinsic Reward）：由于长程任务的外部奖励信号非常稀疏，研究者正致力于开发模型的内在激励机制，让模型基于“好奇心”或“成就感”主动提出假设并自行验证。

3、 从 DeepSeek R1 开始，社区在可验证任务（如 coding、math）上通过 RL scaling 取得了很大成功。但还有两个值得关注的方向：

• 多模态感知：即使是可验证任务，模型能感知更多模态（如图像）也能解决更多真实世界问题。比如小红书团队较早就在做类似 OpenAI o3 的“thinking with image”工作。

• Thinking Efficiency：当前 thinking model 和 chat model 存在割裂，adaptive thinking 往往以牺牲性能换效率，但可以在 RL 阶段做学习，让模型达到相对帕累托最优状态，即 think 越多、结果越好。

4、 Scaling Law 在国内依然非常能打，还没撞墙。 最近国内几个大动作，无论是 GLM-5 把参数翻倍，还是 Kimi 2.5 把数据集翻倍，最后都带来了显著的智能收益。

5、 数据的重要性需要重新提权，数据获取、清洗、长尾挖掘、难例构造与评测闭环，会成为未来模型能力差距的分野点。Seedance 2.0 之所以能拉开差距，数据是绕不开的核心变量。Model card 信息显示它在长尾数据上做了大量精细化处理。DeepMind 等顶级实验室的趋势也表明，研发范式正在从 compute-bound 转移到 data-bound：算力仍是瓶颈，但它会越来越可得，边际重要性下降；真正拉开差距的点转为数据侧。

6、 数据采集是很 labor-intensive 的活儿，而中国在劳动力组织、执行与规模化采集上其实是更有优势的。尤其视频数据，国内存在大量独立运作、极其灵活的视频拍摄团队，甚至是那种没建公司的草根团队，他们拍摄独家视频数据的能力极强。虽然谷歌等也花了大钱，但是质量和人力运营效率都比不上。

7、 中国模型公司目前面临的一个现实挑战是基础设施的短板：相比 Anthropic 和 Google，我们在推理速度和工程稳定性上还有差距（Opus 虽然大但并不慢），且受限于卡量，各家推理资源都比较紧缺。

8、 但国内的缺卡现状倒逼中国公司在架构创新上的积极性比硅谷更高，从底层去解决推理效率问题。这一点让很多硅谷 AI Lab 都感到 impressive，他们卡够用了，反而没动力去做效率上的优化。

9、 AI 市场终局不会是搜索业务的 9:1 格局，而更像是电商行业的百花齐放：“既有淘宝、京东、拼多多，也有抖音、小红书以及微商”。只要不犯战略性大错，各家最终都能分化出自己的 Agent 产品和市场地盘，找到属于自己的生态位。

10、 在能力层面，各家模型很快就能拉齐，所以最后真正的分化可能还是在 taste ，比如是做一个极致理性高效的工具，还是一个有人格有个性的伙伴。

11、 AI Coding 至少是一个千亿美金的市场，而 AI 视频的市场空间不一定比 coding 小，因为十几亿普通用户已经每天在抖音 / TikTok 上消耗 2-3 小时了，在 AI 视频能力进化后，这 2-3 小时带来的意义价值可能还会更大。

12、 与 Coding 领域偏赢家通吃的逻辑不同，影像市场具有更好的分化性。各家模型厂商（如可灵、海螺、Veo）可以在分镜表现力、首尾帧、人物一致性等不同维度做打磨，都能找到差异化生存的空间，不会被一两家垄断。

13、 视频模型能力提升，不只会充实 C 端内容供给，大概率还会外溢到具身智能：机器人公司在未来 1–2 年可能出现明显加速，视频模型可以演变成机器人的大脑，赋予其理解现实世界的能力，并加速机器人数据的积累。

14、 软件工程的工作性质已经发生了实质性的变化。现在硅谷一线 AI startup 里，一个程序员已经可以一整天不亲手写一行代码，而只是通过聊天来驱动 Coding Agent 完成实打实的工程构建。

15、 今天 Agent 能力的提升更多还是靠基模驱动，这个现实会对初创公司构成巨大威胁：

• 原本属于应用层的 System Prompt 和编排框架思路，正在被模型厂商直接封装进基座，导致应用层变得越来越薄。

• 现在的前沿闭源公司已经进化到将 Agent 的执行日志、动作序列以及最终是否被用户采纳的数据，打包成多元组放入训练过程中。

• Kimi 2.5 甚至是通过 RL，把类似 “一个 Orchestrator + 一堆 Sub-Agent” 的编排思路直接写进了模型的内生能力里。

话题  *02*

现象级国产模型深度解析

ByteDance：Seedance 2.0

16、 Seedance 2.0 毫无疑问是春节模型的 magic moment，它的出现标志着国产大模型第一次在全球范围内实现领先，且明显在视频生成维度与 Sora 2、Veo 3.1 等海外顶尖视频模型领先一个代际，具体表现在生产效率与可用性的飞跃上，对于非专业创作者来说，这意味着过去需要耗费一整天打磨的视频，现在半小时就能出片。传统影视制作逻辑在 100 倍的效率碾压下将变得毫无意义。

• 效率革命：Seedance 2.0 在生产同等质量视频时的效率提升了 10 到 20 倍。过去需要长达一天甚至更久的创作任务，现在依靠该模型在半小时内即可完成。

• 直出可用性：Seendance 2.0 实现了极高的“一键直出”成功率，视频创作从 “抽卡逻辑” 进化到了 “工业化可用” 阶段。在旧有的视频模型（如 Veo 3.1）中，生成的 8 秒视频往往只有 3 到 4 秒可用，创作者需要频繁“抽卡”。而 Seedance 2.0 单次生成的 15 秒视频在语义理解和镜头衔接上已基本达到商用门槛。

• 专业镜头语言：模型在评价维度上展现了极高的素养，包括精密的语义理解、镜头切换以及专业级别的镜头语言表达。

未来 6 到 12 个月内，视频创作的门槛将彻底崩塌，我们将迎来真正的 “国民级导演” 时代。一个普通人只要花不到 100 元人民币，就有望制作出一部画质堪比 Netflix、时长 30-40 分钟的高质量长视频。整个内容市场的丰富度将呈指数级上升。

17、 技术视角下，Seedance 2.0 的提升可以拆成几层：

• 第一层：信息提取更分层化，不再只抓轮廓、主体、颜色，而是深入到了纹理与材质的微观提取，让画面有了触感。

• 第二层：对微动作、微表情的建模更细，专门引入处理细微动态的边界编码，例如情绪变化下嘴唇的细小运动、鸟类抖动翅膀这类细节。

• 第三层：因果逻辑与物理关系的理解更强，模型能更好理解 “分镜前后” 的叙事与状态流转。

18、 这种供给的无限释放将带来两类机会：

1） 对现有供给的重塑（如影视、游戏、广告）；

2） 解锁过去无法存在的场景，比如实时生成的互动视频、陪伴服务以及全新的社交形态。

19、 Seedance 2.0 出现后，最难受的是做 “上层工具/工作流” 的团队，传统的 “生图（Midjourney）+ 配音（ElevenLabs）+ 剪辑” 的链条式工作流正在被端到端的模型吞没。

20、 字节在视频模型上的领先具有一定的必然性，且大概率会持续领先，这主要体现在几个方面：

• 战略优先级上，Seedance 在字节内部是核心集团战略，而 Sora 在 OpenAI 内部排序相对靠后。

• 组织战力上，字节能做到 “三代同步”（一代做线上优化，一代做主力研发，一代做超前预研），而国内其他厂商通常只能兼顾两代；

• 算力上，视频模型对卡的需求并没有那么夸张；

• 数据上，抖音、快手等平台积累的视频数据与 YouTube 的差距并不呈数量级；

• 合规角度，国内在法规、版权/IP 的约束上，限制更少。

MiniMax M2

21、 做模型最重要的事情是“定义问题”。只有定义清楚任务和目标，才能决定 Benchmark 是什么。公开的 Benchmark 只是对模型能力的“一种观测”，并不代表模型在市场上的真实价值。如果模型刷榜分很高但用户用不起或用不起来，那就没有意义。原因在于：

• 一方面，SWE 等现有 benchmark 主要关注 Bug fix，但真实的开发场景是全栈复杂的（涉及云的配置、数据库、网关、API 调用等）；

• 另一方面，现在 AI 擅长的语言是 Python/TS，但真实的生产环境是多语言的（Golang, C++, Rust 等）。

22、 Minimax 内部构建了一套更全面的基准测试：VIBE Bench，它涵盖软件开发的方方面面，目标是让一个不会编程的小白也能自己开发一个多语言、全栈的 APP，根据内部测试，其新模型在 VIBE 榜单上比 Claude Opus 4.5/4.6 有显著提升。

23、 Minimax M2 基模的 Size (200B 参数 + 10B 激活) 在目前国内外模型厂商中是很特殊的存在，之所以做这样的选择其实从模型商业化落地角度出发，实现性能、速度、成本的平衡，本质也是为了解决顶尖模型普遍存在的“好用但用不起”的痛点，背后是关于 Agent 落地逻辑的思考。

• 并发需求：在实际工作中，用户可能同时并发 5-10 个 Agent 处理任务，模型的响应速度直接决定了工作流的效率。因此，速度和成本与模型性能同样重要。

• Agents 需要全天候运行：Agent 从开发者泛化到普通用户之后意味着 Agent 一定会变成 24 小时不间断工作的个人助理。如果是高频使用，用户无法承担昂贵的 Token 成本，比如今天 Claude 这样的价格放到一个全天候 Agent 上一定是不 affordable 的；

从 Talkie 中得到的 learnings：这类产品受众广、门槛低、对话轮次长，使用密度极高，如果 agentic 产品最终变成一个 247 个人助理，那么它的使用密度就是和 Talkie 差不多，对应的成本也是类似。Talkie 底层的 M2-her 就是在 M2 基模做的 post-traning。

此外，通过 pre-train 阶段的大量实验，加上对硅谷前沿进展的一手了解，多方交叉验证后确认：在这个模型尺寸上既能达到顶级性能，又能提供足够快的速度。Anthropic 的 Claude 也印证了“不需要超大参数也能取得很好效果”这一判断。

OpenClaw 创始人 Peter Steinberger 在 Claude 账号被封后改用 Minimax 模型一定程度上也验证了 Minimax 在 Performance、cost、speed 上找到了一个很好的平衡点。

24、 Anthropic 最值得学习的经验是“Dogfooding”。MiniMax 全公司都在用 Agent Native 的组织方式工作，即优先让 Agent 自己解决问题，如果解决不了，就把它定义成下一个版本的 Benchmark，而这可能是 Anthropic 真正能把模型和产品做深做好的原因，也是真正领先国内的地方。Minimax 已经在要求全员（包括法务、HR 等）用 Agent 工作，通过内部的真实反馈，收集 Benchmark 覆盖不到的边缘场景，甚至是用 Agent 做 GPU 算子优化等等。

智谱 GLM-5

25、 智谱内部有过一个路线选择上的辩论：

1） 科学家路线：曾就在 Reasoning 场景下追求数学、代码竞赛高分，让 Agent 像科学家一样进科学家路线行长程研究，

2） 工程师路线：追求更有经济价值的生产力任务， 聚焦于解决现实工作流中的编程和 Agentic 问题。

最终，团队选择全面对标 Anthropic、押注工程师路线，因为这个路线更贴近看是使用场景，也有经济价值。

26、 Coding/Agent 任务与 Reasoning 任务有本质差异：

• Reasoning 是在有限空间内做 heavy thinking，小模型靠强化学习就能刷高分；

• 而复杂的 code/agent 任务更依赖工程直觉，需要像有经验的工程师一样一眼找到问题根因，用更少的 Token 直接解决问题，而非反复试错。

27、 因此，scaling 基座规模对解决复杂任务非常必要，GLM-5 的目标是在大多数场景下逼近 Claude Opus 的效果。

28、 在 GLM-5 中选择继续 Scaling 也是团队在实验中发现，模型在 coding 任务上随着规模扩大有显著增益，在工程直觉、思路和知识量都明显提升。

29、 GLM-5 总参数为 744B，激活参数约 40B，模型 size 选择的核心思路是：确保模型能够大规模落地生产部署，是结合实际部署条件来决定的。GLM-5 被控制在 4-8 张 H100 能够承载的规模。

30、 为什么在 GLM-5 中引入 DSA 架构？

• 在 Coding 场景中，输入（Context）通常远大于输出（Generation），从算力角度造成了极大压力，为了优化这类 long context coding 场景下的成本，在 GLM-5 引入了 DSA（DeepSeek Sparse Attention ）架构；

• DSA 架构允许模型在处理如 200K 长度的序列时，只关注局部窗口内或经过筛选的有限 token，而不需要关注序列中所有其他 tokens，而从而将计算复杂度降低，使超长序列处理变得可行；

• 在大量消融实验（Ablation Study）中，DSA 架构在保持性能无损的同时，极大地减少了 Prefilling 阶段的计算量，在前面提到的 long context coding 场景下，理论上可以显著压低推理成本。

31、 如果把 coding 市场粗分为高端/中端/初阶三档。Anthropic 目前占据高端市场，享受高溢价。GLM 4.5/4.7 在切中端用户，而 GLM 5 试图挑战中高端。在性能接近的情况下，GLM 约 3 刀的成本对标 Sonnet/Opus 的 15-25 刀，对开发者有极大的吸引力，且国内开源模型对 Claude skills 等生态兼容度很好，切换也相对顺滑。

话题  *03*

Token 消耗、模型商业化与出海

32、 Token 消耗已经成为共识性的新指标，在火山云过去几年对 Token 消耗的追踪中，可以看到模型智能落地场景的变迁：

• 24 年：token 消耗主战场在 research、数据处理等研究类场景，大模型更多是替代人力；

• 24 年-25 年上半年是文本模型时代，互动娱乐、客服/销售、教育几个头部场景三足鼎立，各占 ～15% 左右；

• 25 年中：随着 Seedance 视频模型的发布，视频生成因单次 Query 的 Token 消耗远超文本，迅速成为 Token 消耗的主力。

• 25 年下半年至今：Agentic 场景异军突起，占比直接超过互动娱乐，冲到了 35%，预计 2026 年这块能占到一半，工具属性彻底压倒娱乐属性。

33、 对 2026 年的 Token 消耗增长预判：

• 在开源模型框架的成熟、Coding 模型能力的提升以及推理成本的持续下降的带动下，2026 年仅 API 商业化一项的 Token 消耗将实现 7 到 10 倍的同比增长，bull case 还会更高。

• 多模态的拉动效应：视频生成模型（如 Seedance 2.0）的普及对 Token 消耗产生了巨大的杠杆作用。单次视频生成 query 消耗的 Token 量远超文本数据处理，这直接改变了流量结构的分布。

2025 年上半年大家还在玩文本和简单 Agent，下半年视频模型才刚发力。而到了 2026 年，开局就是 Seedance 2.0、OpenClaw，年中可能还会有新的长程思考模型和异步 Agent 入场。再加上各行各业都带着实体场景在推出新产品，这一轮增长是全产业维度的，增长十几倍完全可能。

34、 但需要注意的是，在多模态场景下，因为使用场景的高度分化，Token 消耗不太能作为北极星指标存在。

35、 国产模型目前普遍采用低价竞争策略以换取市场份额有可能会把市场带入“中国制造业式的内卷”：

• 价格差对比：在处理复杂的 Coding 任务时，国产 GLM 等模型的输出价格约为 3 刀（每百万 Token），而 Anthropic 的 Sonnet 4.5 定价为 15 刀，Opus 系列（如 4.5/4.6）则高达 25 刀。

• 像光伏、锂电、电动车一样，全世界都在用中国制造，虽然技术处于世界一流，但由于全行业极度“内卷”导致毛利或净利微薄，除了极少数头部企业（如模型界的“宁德时代”），其他厂商难以维持健康的盈利水平。

• 在地缘竞争的背景下，不排除未来海外会 block 掉国内模型（比如以蒸馏为由限制国内模型的分发），也有可能会像电动车一样征收 100%、200% 的高额关税。

36、 商业化转正的关键在于推理效率的持续优化：

• 通过推理 Infra 的优化，确保每一次 query 的推理成本为正，依靠规模效应实现毛利；

• 深度定制：帮助企业进行 Post-training 或 RL ，精细化构建企业私有数据与模型壁垒，也是高净值客户的核心需求。

37、 受地缘政治和中美关系影响，国产模型直接向美国企业提供 API 服务存在数据隐私和合规方面的天然劣势。在这种限制下，围绕 License 仍存在几种创新的出海路径：

• 策略 1 ：借壳“OpenRouter 生态”出海：

今天在中国开源模型生态下真正赚到钱的是美国的 Neo-cloud 和 MaaS 平台，中国模型虽然无法直接在海外做商业化，但其技术价值正在被海外中间商大量变现：

1）    ：可以简单理解为视频模型的 OpenRouter，估值已经达到 45 亿美金，平台上的核心模型很多都是中国的（可灵、海螺、Seedance...）；

2） Fireworks：通过优化部署中国开源模型（如 DeepSeek、千问等）同样在海外市场赚取了丰厚利润。

• 策略 2：“延迟开源”

在每一代模型训练完成后，先设立为期 3-6 个月的 Preview 期，仅授权给选定的海外合作伙伴（如特定的 Cloud Provider）使用，并按使用量或 license 计费，只有当下一代模型（V7）发布时，才将上一代模型开源。这样既能持续拱高全球开源地基以威胁 Anthropic 等闭源厂商的份额，又能确保厂商在每一代前沿模型上都能收到海外市场的 revenue share。

38、 在 Coding/Agentic 能力上，模型出海面临截然不同的三个战场：

• 个人开发者：很容易切，并且已经拿下了很大一块份额；

• Agent 工具商：Cursor 这类的应用层也非常有动力切换成中国模型，因为他们不想永远给 Anthropic 打工，只有通过“混用模型”把成本降下来，他们自己的商业模式才能真正跑通；

• 中大 B：最难啃的骨头。美国大企业对成本不敏感，但只在乎两件事：Performance 和 Security。中国模型虽然在模型能力和数据安全上做得很到位，但会因为地缘政治原因无法获得美国企业主的信任。

39、 企业端对 Agent 的付费在今年大概率会有明显提升，有四个核心逻辑支撑：

• 部署侧 CapEx 优化：以前模型越大部署越贵，但目前通过量化感知训练，以及像 DeepSeek Conditional Memory 这类技术，可以把部分参数直接加载在廉价的 SSD (固态硬盘 ）里，而不是昂贵的 HBM（高带宽内存）。这意味着企业做私有化部署时，或者买一体机时，在普通企业级一体机上部署超大模型成为可能，大幅降低了私有化部署的 CapEx；

• 评估环节更贴合实际场景：Seed 2.0 直接拿用户真实场景的 Query 类型和典型场景去做评估，甚至拿回来做训练。这意味着模型会越来越贴合下游的具体需求，而且在实际模型应用中会发现，企业高频需求的场景其实并没有那么发散，还是比较集中的，所以模型厂商可以快速跟进企业现实需求；

• 办公场景的深度渗透：1）一方面今年各家模型厂商都在全力进攻办公软件场景，这不仅仅局限于 Office 套件，还包括各种垂类 SaaS 软件，2）另一方面因为多模态视觉理解能力的提升，模型对高分辨率图像的理解不断加强意味着模型可以直接看懂、甚至操作各种复杂的办公软件，且这类对 UI 理解已经成为公开打榜方向。

• 基模降价：在去年 Claude 4.5 是 Anthropic 出现了首次价格下调，这种降价并非主动让利，而是市场竞争（如 Google 的 Gemini Pro、OpenAI 的 GPT 5.2 及 Codex）倒逼顶尖模型厂商做出价格下探。到 2026 年，开源模型能力的提升会进一步冲击 Anthropic 这些闭源模型的溢价空间。

40、 国民级的产品落地能力也是国内厂商的独特优势：海外产品倾向于服务高精尖用户，用最强的模型做生产力产出，而国内产品，比如豆包，则致力于把智能包进生活场景里，让老人、小孩等普罗大众低门槛地接入 AI。豆包大概 80% 的用户行为都发生在跟 Agent (而非 Chatbot) 的交互中，包括作业答疑、植物识别、逛博物馆等等。

41、 长期看 ToB 赚取的是辛苦钱，ToC 赚取的才是生态位：

• Seedance 2.0 的出现既打开了 ToC 的想象空间，也引出一个战略反思：过去两年，中国模型公司把 “多模态和娱乐” 的优先级降低，转而去卷生产力场景，如果拉长到五年的维度来看，是不是一个战略失误？

• 如果要最大化利润，ToB 面临的不确定性（关税、中间商、价格战）太多； Seedance 2.0 让人看到了 “再造几个大 C 端平台” 的机会，且 ToC 的娱乐化场景监管更少、限制更小，可能才是中国公司最可能绕开地缘政治压制去打全球市场的路径；

• 历史上中国真正做成的全球化产品也全是 ToC 的（TikTok、Temu、Shein，东南亚的 Shopee 等）。

• 长期看，中国厂商的优势不在于直接卖 API，而在于产品出海。拼创意、拼迭代速度，以及拼中国特有的电力成本优势。

42、 出海的重心不必绑定美国市场。考虑到地缘政治因素，美国市场可能难以切入，但除此之外的全球市场机会相当广阔。
