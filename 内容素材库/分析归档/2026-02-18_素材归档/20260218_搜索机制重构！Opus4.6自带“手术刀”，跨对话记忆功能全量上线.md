今天发布的Sonnet 4.6版本中，一项被低估的开发者升级：

Claude 的网络搜索和获取工具现在会在结果到达上下文窗口之前编写和执行代码来过滤结果

核心看点在于全新的动态过滤机制。

它彻底改变了 AI 搜索网页的逻辑：让模型在搜索过程中原生编写并执行代码，先把网页内容清洗一遍，再喂给上下文窗口。

效果立竿见影：搜索任务准确率平均提升 11% ，同时输入 Token 消耗降低 24% 。

以前是大力出奇迹，现在是精准手术刀。

传统的 AI 搜索是个极其耗费 Token 的苦力活。

Agent 通常需要发起查询、抓取大量网页 HTML、全部塞进上下文，然后再进行推理。

这种方式不仅贵，而且网页中的广告、导航栏等各种噪声会严重干扰模型判断，导致幻觉产生。

动态过滤机制就是为了解决这个问题。

现在的 Claude 不再无脑阅读完整的 HTML 文件。

它会自动编写 Python 代码对查询结果进行后处理，动态筛选出相关信息，直接丢弃无关数据。

这意味着进入上下文窗口的，全是干货。

数据表现相当硬核。

在 BrowseComp 基准测试中，这项技术让模型在复杂网页中寻找针尖般信息的各个能力大幅跃升。

Sonnet 4.6 的准确率从33.3%升至 46.6%。

更强的Opus 4.6则直接从45.3%干到了 61.6%。

在另一项针对多答案研究任务的 DeepsearchQA测试中，Opus 4.6的 F1 分数达到了 77.3%，不仅找得准，而且找得全。

Quora旗下的 Poe 团队已经在内部实测了这项功能。

他们的反馈：Opus 4.6 表现得像一个真正的研究员，它是在写 Python 代码来解析、过滤和交叉验证结果，而不是对着原始 HTML 发呆。

目前，这项功能已向开发者开放。

在 API 中调用 Sonnet 4.6 或 Opus 4.6 时，该功能默认开启。

开发者只需在 Header 中添加  `anthropic-beta: code-execution-web-tools-2026-02-09`  即可体验。

```python
// Header: anthropic-beta: code-execution-web-tools-2026-02-09  
{  
  "model": "claude-opus-4-6",  
  "max_tokens": 4096,  
  "tools": [  
    {  
      "type": "web_search_20260209",  
      "name": "web_search"  
    },  
    {  
      "type": "web_fetch_20260209",  
      "name": "web_fetch"  
    }  
  ],  
  "messages": [  
    {  
      "role": "user",  
      "content": "Search for the current prices of AAPL and GOOGL, then calculate which has a better P/E ratio."  
    }  
  ]  
}
```

除了动态搜索，Anthropic 这次还把工具库做了一次大扫除。

代码执行、记忆功能、编程工具调用全部宣布全面开放。

代码执行：提供沙盒环境，让 Agent 在对话中直接运行代码。无论是过滤上下文、分析数据还是进行复杂计算，都能实时搞定。

记忆功能：利用持久化文件目录，实现跨对话的信息存取。让 Agent 摆脱“金鱼记忆”，保留上下文的同时，不再占用宝贵的上下文窗口额度。

编程工具调用：通过代码编排复杂的多工具工作流。中间步骤产生的冗余数据直接被隔离，不再污染上下文窗口。

工具搜索：支持从海量工具库中动态检索。再也不用把成百上千个工具定义一次性全部塞进 Prompt 里了。

工具使用示例：在工具定义中直接预埋调用范例。手把手教模型正确姿势，大幅降低参数填错的概率

尤其是 Memory功能，允许 Agent 通过持久化文件目录跨对话存储信息，不再受限于单次上下文窗口。
