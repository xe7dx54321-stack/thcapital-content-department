## 01

## OpenClaw 的出现，

## 带来了 4 个层面的爆发式增长

OpenClaw 虽然前期安装麻烦，但是带来了全新的体验，包括自动化交付、长记忆、可自定义的技能、本地化操控等。带来了哪些爆发性增长？答案就是：模型、Skill、MacMini 和数据库等 Infra。

模型层，营收狂飙

所有的模型厂商都在享受 OpenClaw 的这一波红利。Kimi 模型 API 在短短 20 天内所创造的收入，就已经超过了整个 2025 年的全年收入。

硬件载体溢价

作为本地化部署的载体，MacMini 在二手市场的价格走势令人咋舌。在闲鱼平台上，2 月初的成交价大约还在 1700 元左右，但到了现在，价格已经暴涨到了 3300 元。价格的飙升速度，甚至超过了以往 GPU 的涨价幅度。

Skill、 Agent 指数级增长

Skill： 节前 Clawhub 上的 Skill 数量大约在 5000 多个，现在已经迅速飙升到了 11232 个；

Agent： 根据 Moltbook 平台的数据，最新的 Agent 注册数目到了 2846423 个，我依稀记得，这个数字在春节前只有 50 多万。

Infra 层重构

记忆模块、数据库等基础设施正在经历与前端应用同步的爆发。比如，Supabase 官方就明确表示，正在经历爆发性的业务增长，因为 agent 正在将 Supabase 作为默认工具来设置和托管 Postgres 数据库。

除了数据库，安全层也是，衍生出了沙盒托管。开发者可以在 OpenClaw 和外部数字世界之间设立一道防火墙，扫描所有进出的数据流量，来有效拦截敏感信息的泄露、恶意的指令注入以及有害代码的执行。

## 02

## OpenClaw，正在成为 AI 时代的 Linux

很多人说 Agent 编排层是一个陷阱。

The Information 分析："所有人都在抢编排层，但这一层会在 18 到 24 个月内被模型厂商吸收。OpenAI 的 Operator、Anthropic 的 Claude Cowork——这些都是模型厂商在收回编排层。

但 OpenClaw 既是也不是，因为这是第三方且最火的编排层。所以 OpenClaw 不是像一个操作系统，而就是这个 AI 时代的 Linux。

我借助豆包梳理了一下 Linux 的历史，毕竟对大部分人来讲有点久远了。首先是上世纪 90 年代初，服务器市场的格局清晰又固化：

- 「贵族选手」UNIX： 由 Sun、HP、IBM 等厂商主导，性能强、稳定性高，但价格贵到离谱，一套系统动辄几万甚至几十万美元，还不兼容其他厂商的硬件和软件，升级还要额外付费。
- 「野心选手」Windows NT： 微软刚推出的服务器系统，试图抢占 UNIX 的市场，但初期功能不完善，稳定性堪忧。
- 「理想选手」GNU： 自由软件运动的核心，有完整的工具链（比如 GCC 编译器、Bash 终端），但缺一个能用的内核，始终无法形成完整的操作系统。

就在这时，芬兰大学生林纳斯·托瓦兹（Linus Torvalds），出于「想在自己的 x86 电脑上跑类 UNIX 系统」的简单想法，写出了 Linux 内核。最初的 Linux，没有华丽的界面，没有完善的功能，甚至连图形化都没有，只是一个能在廉价 PC 上运行的、免费的类 UNIX 内核——谁也没料到，这个「小玩具」，后来会颠覆整个行业。

第一波： UNIX 的「致命缺点」，恰恰是 Linux 的「天然优势」。当时的互联网刚起步，大量初创公司、小网站需要服务器，但根本用不起昂贵的 UNIX 系统。而 Linux 刚好戳中了所有痛点：

- 完全免费，无任何授权费用，初创公司零成本就能使用；
- 兼容廉价的 x86 PC 服务器，不用为专用硬件花钱；
- 开源特性，全球开发者一起迭代，漏洞修复快、功能更新快；
- 发行版（如 Debian、Red Hat）统一兼容，不锁死硬件，迁移成本极低。Linux 快速吞噬了 UNIX 的低端市场——小公司、个人站长、初创互联网企业，几乎清一色选择 Linux。

等 UNIX 厂商反应过来时，Linux 已经站稳脚跟，开始向中高端市场渗透。这一轮，Linux 赢在「性价比」，更赢在「开源协作的速度」。

第二波： Linux 最狠的地方，不是单一系统强，而是打造了一套「全免费、开箱即用」的建站解决方案——LAMP：Linux（系统）+ Apache（web 服务器）+ MySQL（数据库）+ PHP/Perl/Python（编程语言）。

一套几十美元的普通 PC 服务器，装上 LAMP，就能搭建一个稳定运行的网站。于是，90 年代末到 2000 年代初，全球 80% 的网站都跑在 LAMP 架构上——包括 Google、Facebook、Amazon 这些后来的互联网巨头，早期全是 Linux 的「忠实用户」。这一轮，Linux 赢在了「生态闭环」，彻底锁定了互联网的基本盘。

第三波： 互联网公司的需求越复杂、规模越大，就越依赖 Linux。随着用户量的增长，Linux 的生态也越来越完善——更多开发者贡献代码，更多厂商适配硬件，更多软件支持 Linux，形成了「用户越多→生态越强→用户更多」的正向循环。对比 Windows，Linux 全是「加分项」：授权免费、几乎不宕机、开源可定制、脚本一键自动化运维。服务器、云端、嵌入式、移动设备方面，Linux 全面碾压，无可替代。Windows 退守桌面端。

第四波： 容器和云原生时代，Docker、Kubernetes（K8s）的出现，彻底关上了 Windows 服务器追赶的大门。云原生时代，从开发、测试到部署、运维，全流程都基于 Linux 环境：容器化部署、微服务架构、自动化运维工具，几乎全是为 Linux 量身打造。所以我们每天接触的一切数字服务，背后都有 Linux 的身影：

- 🌐 所有网站、APP 的后端服务器；
- ☁️ 阿里云、腾讯云、AWS 等所有云服务；
- 📱 全球 90% 以上的智能手机（Android 基于 Linux 内核）；
- 🏠 路由器、NAS、智能电视、智能家居；
- 💻 超级计算机、AI 训练集群、自动驾驶系统。

OpenClaw 为什 么和 Linux 相似？

1. 和 Linux 一样，OpenClaw 以开源为基石；
2. Linux 靠「内核 + 发行版」实现灵活适配，OpenClaw 则通过「核心框架 + Skill 插件」实现功能扩展；
3. 同样以社区为核心，吸引开发者贡献技能插件，同时已有硬件厂商推出预装 OpenClaw 的设备，形成「社区 + 商业」的正向循环；
4. 同样轻量化 + 可定制，适配多元场景，兼顾个人用户与企业需求。

所以未来的生态发展可以想到，在硬件、应用和底层 Infra 上，都会有 Agent-Native 的生态逐渐出来。

## 03

## OpenClaw 正在成为典型的交互入口，

## 短期内可能爆发这些机会

春节前总结了下最近一年的新 AI 公司基本覆盖在：交互入口、数据入口、载体入口，有两两交叉的，但三个都有是大厂生意（比如眼镜）。

OpenClaw 可能成为典型的交互入口， 除了可能作为个人对外的入口，还具备高度的自动化能力，从而对其他交互入口有不可替代的体验差和包容性，成为入口中的入口。

短期可能有哪些生态位机会？

通信

理论上你可以让你的 OpenClaw 注册一个 Gmail 账户来使用邮件，但实际上这非常困难，因为 Gmail 和所有邮件提供商都故意让自动化使用变得尽可能困难，以防止垃圾邮件。当然也可以让 OpenClaw 在飞书、discord、Slack、Whatsapp 甚至企业微信去传输信息和文件，但如果用户逐渐习惯于以一个对话框的形式去做，其他通信工具会被替代掉。我自己的体感就是让 OpenClaw 使用工具是不舒服的。

所以可能需要一个 AI-Native 的通信系统，例如邮件系统，未来可能成为 Agent 版本的微信。这也是我自己手搓分布式通信 Skill 的初衷（虽然不太好用）。分布式是一定的，因为大部分 OpenClaw 目前都在本地端，未来可能会端云结合到最终云端为主（没有带本地存储文件的必要了）。人-Agent 通信与 Agent 间通信还有很长的路要走。

群聊和协作

通信之后一定会开始解决协作问题。上面的通信是点对点，之后就会出现群聊，因为目前让 OpenClaw 在飞书、discord、Slack、Whatsapp 甚至企业微信进行人机协作，甚至在群里加入几个 Bot，都很难实现复杂任务，因为操作本身受限于飞书等软件的权限，这些也不是 AI-Native 的。

群聊和协作也会有 Multi-Agent 的组织架构问题。例如已经有开发者以管理者-执行者的伞状架构建立自己的 AI 团队，也有阿米巴式的架构来解决更加复杂的问题。需要可以模块化、可评价和进化的框架来为 OpenClaw 升级。高度的自动化会带来新的管理方式的问题，包括组织进化、职能分配如何在自治+监督的 Multi-Agent 框架下做好。有的人看到用的 OpenClaw 因为发现有事情卡在人那里，自己跑来催主人约 1v1 会议或者帮主人发邮件约会。所以人和 AI 的管理关系开始反转，如果人人都有 Agent 后更是。另一个问题是现有 SaaS 的工作流会被 Agent 重构，一部分功能会被 AI 托管，被原子化到每个人的 OpenClaw 里。

交易市场

我们最近几年都在研究 AI 交易平台，看到从清单到标准服务交易的平台已经走到了需要 AI 来撮合非标的时代，因为分散、非标准化且不易被清晰描述的服务产品对 AI 来讲理解和撮合反而是容易的。但随着 OpenClaw 带来的交互入口改变，可能之前一部分低频标准化服务也会受到影响，如酒旅，只剩下外卖、打车等高频标准化服务。

从另一方面来看，Agent 本身就具备在数字化领域的服务能力，Agent 的交易市场也会更加顺理成章，因为 A2A 在沟通和需求匹配上更加容易，只需要解决安全、定价和支付等问题。从需求角度讲，Agent 帮人赚钱是排名最靠前的需求之一。在人本身技能与 AI 无法融合的时候，AI 就是人的经纪人。人本身与 AI 技能融合，AI 就是能力的放大器。

物理世界

用户有给 OpenClaw 接入 ROS2 机器人系统的框架，让 OpenClaw 跑在 Apple Watch 上，还有给 OpenClaw 加了视觉和障碍检测。在具身机器人遍布工厂和家庭之前，视觉、语音这些交互方式会天然和 OpenClaw 这样的交互入口融合，缺的就是一个载体。比如现在程序员都在用语音输入法接入 Vibe Coding，因为本身语音就是更快的输入方式。

载体可以是摄像头、手环手表或者机器在人，也可以是音箱。如果没有多模态入口，就是被调用的硬件而不是载体了。在这点上很像 Linux，吃掉了数字硬件的大部分市场。
